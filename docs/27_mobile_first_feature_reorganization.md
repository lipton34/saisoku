# モバイル中心の機能再編・改修仕様

最終更新: 2026-07-25

## 1. この文書の目的

この文書は、Saisokuの機能再編とモバイル向け改修について、議題、決定事項、未決事項、実装要件を一元管理するための正本である。

検討は議題を一つずつ進める。各議題について、質問と追加確認事項をすべて解決し、決定内容をこの文書へ反映してから次の議題へ移る。

最終的には、この文書だけで改修対象、期待動作、データ移行、権限、画面構成、検証内容を判断して実装を開始できる状態にする。

## 2. 検討の進め方

1. 「議題一覧」の順番で一つの議題だけを扱う。
2. 選択肢、影響範囲、推奨案を確認する。
3. 回答から発生した質問と追加確認事項を解決する。
4. 決定内容を「確定事項」と対象機能の仕様へ記録する。
5. 対象議題を `確定` にしてから次へ進む。
6. 全議題の確定後、要件間の矛盾と実装漏れを横断確認する。
7. 実装開始条件をすべて満たした後に改修作業へ移る。

検討中の内容を、確定した仕様として実装しない。

## 3. 前提

- 利用者は固定メンバー5人である。
- 主な閲覧・操作環境はスマートフォンなどの携帯端末である。
- 現在の画面はPC利用を前提とした情報量とレイアウトが多く、スマートフォンで見づらい。
- GBF本体で完結する機能は利用頻度が低い。
- SaisokuはGBF本体の代替ではなく、団内での共有、相談、記録、周回補助に価値を置く。
- 既存機能は即座に削除するだけでなく、簡略化、統合、保管庫への移動による活用も検討する。
- 既存の認証、所有者判定、ユーザーデータを維持する。
- 新しい画面構成は360px幅を基準にし、768pxおよび1280px以上でも利用可能にする。

## 4. 現時点の確定事項

### 4.1 機能の分類

主メニューに残す機能:

- ホーム
- 周回目標（現在の素材目標管理）
- 進捗目標
- 古戦場目標計算
- 編成共有
- 目標管理（現在の団内目標管理）

保管庫へ移す機能:

- イベント予定
- 公式NEWS

完全に削除する機能:

- ロードマップ
- 個人タスク管理
- 旧ツール・プレースホルダー

利用者向けメニューへ表示しない内部基盤:

- 認証
- GBFデータマスタ
- 画像保存
- APIおよびDB基盤

### 4.2 周回目標

- 素材目標管理の利用者向け名称を「周回目標」へ変更する。
- 進捗目標とは統合しない。
- 素材数だけでなく、討伐回数や周回数などの数値目標にも使用する。
- 進捗目標より手軽に作成、確認、更新できる独立機能とする。

### 4.3 目標管理

- 現在の団内目標カンバンをホームの中心機能として表示する。
- 団内目標と個人目標をタブで分ける。
- 目標作成時に団内公開または個人用を選択できるようにする。
- 個人目標は完全非公開とし、作成者以外には存在自体を返さない。
- 画面上の非表示だけに頼らず、APIの取得、更新、削除で所有者と公開範囲を検証する。

### 4.4 編成共有

- キャラクター、武器、召喚石、ジョブなどを個別選択して編成を作成する手順を廃止する。
- 画像投稿を基本手順とする。
- 画像に収まらない情報や補足を入力するテキストフォームを用意する。
- 選択式の詳細入力は任意機能としても残さない。
- この方針は `docs/01_overall_policy.md` の現行方針と衝突するため、実装前に全体方針と編成関連要件を正式に更新する。

### 議題1: 共通ナビゲーション

状態: 確定

目的:

- スマートフォンを主対象として、普段使う機能へ少ない操作で移動できる構成にする。
- 主ナビゲーションへ表示する情報を減らし、低頻度機能を保管庫へ整理する。

現状:

- PC向けの左サイドバーに、ホーム、タスク、素材、進捗、目標、古戦場、NEWS、イベント、編成一覧、編成投稿、ロードマップが並んでいる。
- スマートフォンでもサイドバーを変形したナビゲーションを使用している。
- ホームとログアウトはサイドバー内にある。

確定事項:

- すべての画面幅で同じ下部ナビゲーションを使用する。
- 下部ナビゲーションの並び順は次のとおりとする。
  1. 周回目標
  2. 進捗目標
  3. 編成
  4. 古戦場
  5. 保管庫
- 下部ナビゲーションの各項目には、用途を判別できる専用アイコンを表示する。
- 既存のアイコンに用途の合うものがあれば流用し、不足する場合だけ新しいアイコンを選定する。
- 「その他」は設けない。
- 保管庫は専用画面とする。
- 古戦場は開催時期に関係なく常時アクセス可能にする。
- 目標管理は独立した主メニュー項目として残さず、ホームへ集約する。
- ホームボタン、ユーザー情報、ログアウトボタンはヘッダーへ移す。
- ヘッダーは下方向へのスクロールで隠し、上方向へのスクロールで再表示する。ページ最上部では常に表示する。
- ヘッダーの左側にホームアイコンと「Saisoku」を表示する。
- ヘッダーの右側にユーザー名とログアウトアイコンを表示する。
- スマートフォンで幅が不足する場合はユーザー名を省略し、アイコンだけを表示する。

未決事項:

- なし。

質問・追加確認:

- すべて解消済み。

影響する画面:

- 認証後の全画面
- ホーム
- 保管庫の新規画面

影響するAPI:

- ナビゲーション自体によるAPI変更はない。
- ログアウトは既存の認証APIを継続利用する。

影響するデータ:

- なし。

移行・互換性:

- 既存ルートの変更有無は、各機能およびデータ移行の議題で決める。
- 保存済みURLから各画面へ直接アクセスできる構成を維持する。

スマートフォン要件:

- 360px幅で5項目の下部ナビゲーションが収まり、ラベルとアイコンを判別できること。
- 下部ナビゲーションが本文の操作や最終行を隠さないこと。
- ヘッダーが非表示でも、主要5項目間を移動できること。
- ログアウトは誤操作しにくく、再表示可能な場所へ置くこと。

完了条件:

- 達成済み。

### 議題2: 共通モバイルレイアウト

状態: 確定

目的:

- 360px幅を基準に、情報量が多すぎず、主要な閲覧と操作を迷わず行える共通レイアウトを定める。
- 画面ごとに異なる操作方法を増やさず、タブ、カード、作成操作、ダイアログの扱いを統一する。
- 768pxおよび1280px以上でも、スマートフォンと同じ機能と操作順を維持しながら余白を活用する。

現状:

- PC向けに複数列、横長のカンバン、多数のボタンや説明を同時表示する画面がある。
- スマートフォン向けの調整はあるが、画面ごとに情報密度と操作方法が異なる。
- 作成・編集にダイアログとページ内フォームが混在している。

確定事項:

- スマートフォンなどの携帯端末を主対象とする。
- 最小対応幅は320pxとし、360px、768px、1280px以上で確認する。
- すべての画面幅で同じ下部ナビゲーションを使用する。
- 各画面の主要な新規作成ボタンは、下部ナビゲーションと重ならない画面右下へ固定表示する。
- 作成機能がない画面では、固定の新規作成ボタンを表示しない。
- タブは画面上部へ横並びで配置し、横幅に収まらない場合はタブ領域内だけ横スクロール可能にする。
- 入力項目が少ない作成・編集はダイアログを使用し、入力項目が多い場合は専用画面へ移動する。
- ダイアログと専用画面を分ける目安は入力3項目程度とし、入力内容や操作の複雑さに応じて個別機能の議題で最終判断する。
- PC・タブレットでも基本的な画面構造は変えず、本文に最大幅を設定し、余裕がある一覧だけ複数列にする。
- PC・タブレットでの具体的な最大幅と列数は、各画面の改修後に実表示を確認して決める。
- 説明、完了済み情報、補助操作などの優先度が低い情報は初期表示を簡潔にし、利用者の展開操作によって表示する。
- 一覧画面は、判断に必要な要点だけをカードへ表示し、カードをタップすると詳細画面へ移動する形式を共通の基本方針とする。
- 一覧内で素早く更新する価値がある操作は、個別機能の議題で例外として定める。

未決事項:

- なし。

質問・追加確認:

- すべて解消済み。

影響する画面:

- 認証後の全画面
- 共通ヘッダー
- 共通下部ナビゲーション
- 一覧、カード、タブ、フォーム、ダイアログを使用する各画面

影響するAPI:

- なし。個別機能の表示件数やページングが必要な場合は、該当議題で決める。

影響するデータ:

- なし。

移行・互換性:

- 既存データは変更しない。
- 既存画面の操作導線は変わる可能性があるが、各機能の実データへ引き続きアクセスできるようにする。

スマートフォン要件:

- 主要な閲覧と操作を縦方向の流れで完結できること。
- ページ全体に意図しない横スクロールを発生させないこと。
- 下部ナビゲーション、固定操作、入力中のキーボードがコンテンツを隠さないこと。
- 主要なタップ対象は可能な範囲で約44px以上を確保すること。
- 長い日本語、URL、ユーザー入力がカードや画面幅を押し広げないこと。

完了条件:

- 達成済み。

### 議題4: ホームの役割

状態: 確定

目的:

- ホームを目標確認と目標操作の中心画面にする。
- 下部ナビゲーションと重複する機能一覧や、保管庫へ移す個人タスクの情報を整理する。
- アプリを開いた直後に、団内目標または個人目標を確認できるようにする。

現状:

- 今日やることを示す大きな案内領域がある。
- 優先タスク5件と団内目標5件を表示している。
- 実装済み機能への大きなカードリンク一覧を表示している。
- 目標の全カンバンは独立した `/goals` 画面にある。

確定事項:

- 現在の目標カンバンをホームの中心機能として表示する。
- 団内目標と個人目標をタブで分ける。
- 目標管理を独立した主メニュー項目として残さない。
- 主要機能への移動は共通の下部ナビゲーションで行う。
- ホームの基本構成は「目標」の見出し、団内目標・個人目標タブ、目標カンバン、右下の目標作成ボタンとする。
- 現在の大きな案内領域、優先タスク、機能カード一覧はホームから削除する。
- 初期表示するタブは「団内目標」とする。
- 編成、周回目標、進捗管理、古戦場の要約はホームへ追加しない。
- ホーム上で目標の閲覧、タブ切り替え、新規作成、状態変更を行えるようにする。
- 詳しい編集と関連情報の管理は、目標カードから目標詳細へ移動して行う。
- 周回目標および進捗管理の項目を、ホームの目標ボードと連携できるようにする。
- 連携した項目のボード上の状態は、目標ボード側で変更できるようにする。
- 新規目標の公開範囲は個人目標を初期値とする。
- 個人目標は、確認メッセージを表示したうえで団内目標へ変更できる。
- 団内目標から個人目標への変更は禁止する。
- 周回目標・進捗管理の作成時に「目標ボードに表示」を初期ONで表示し、利用者が解除できるようにする。
- 周回目標・進捗管理から作成する連携カードは、個人目標として追加する。
- 連携カードへ値を複製せず、元の周回目標・進捗管理を正本として参照する。
- 元の周回目標・進捗管理を削除した場合、連携カードも削除する。
- 周回目標の数値は周回目標画面と目標カードの両方から変更でき、同じ値へ反映する。
- 周回目標のカード本体を押すと周回目標画面へ移動する。
- 進捗管理の内容変更は目標カード内で行わず、カード本体から進捗管理画面へ移動して行う。
- 既存の「今やる」「次にやる」「後でやる」「保留」「完了」とは別に、初期状態用の「未設定」を先頭へ追加する。
- 通常作成した目標と連携カードを含む、すべての新規目標を「未設定」から開始する。
