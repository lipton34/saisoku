# 団内目標共有機能 要件定義書 v0.1

## 1. この文書の目的

この文書は、saisoku に追加する「団内目標共有機能」の要件を整理するための文書である。

本機能は、団員ごとに目標を設定し、その進捗を団内で共有できるようにすることを目的とする。
また、団員同士で目標を提案できるようにし、提案された本人が受け入れた場合のみ、その目標を自分の目標として登録できるようにする。

この段階では、詳細なDB設計や実装コードには踏み込みすぎず、既存の saisoku 全体方針に合う要件定義として整理する。

---

## 2. 機能の位置づけ

団内目標共有機能は、古戦場専用ではなく、団内で使う汎用的な個人目標共有機能とする。

ただし、最初の利用イメージとしては、以下のようなグラブルの団内活動を想定する。

```text
古戦場の個人目標
高難度挑戦に向けた準備目標
レヴァンス・高難度周回の武器集め目標
キャラ・武器・召喚石などの育成目標
団内で共有したい個人的な取り組み目標
```

本機能は、単なるノルマ管理ではなく、団員が現在何を目標にしているか、どこまで進んでいるかを団内で確認しやすくするための共有機能として扱う。

---

## 3. 想定ユーザー

## 3.1 目標を設定する団員

自分の目標を登録し、進捗を更新するユーザー。

例：

```text
古戦場で個人10億を目指す
シエテ剣を5本集める
ルシゼロに参加できる編成を作る
次の古戦場までに肉を8000個集める
```

## 3.2 他団員の目標を確認する団員

団員全体の目標や進捗を確認するユーザー。

例：

```text
誰がどの高難度に向けて準備しているか確認する
古戦場前に各団員の目標を確認する
武器集めや編成準備の進み具合を見る
```

## 3.3 目標を提案する団員

他の団員に対して、取り組んでほしい目標を提案するユーザー。

例：

```text
Aさんに「天元HL用の水編成を作ってみない？」と提案する
Bさんに「次の古戦場で個人10億を目指してみない？」と提案する
Cさんに「シエテ剣をあと2本集めてみない？」と提案する
```

---

## 4. MVPで実現すること

MVPでは、以下を実現する。

```text
自分の目標作成
自分の目標進捗更新
団員全体の目標一覧表示
団員・カテゴリ・状態・期限・キーワードによるフィルター
目標詳細表示
他団員への目標提案
自分に届いた提案一覧
提案の受け入れ
提案の見送り
受け入れた提案から個人目標を作成
```

現在は1つの団での運用のみを想定する。
そのため、複数団管理や複雑な権限管理はMVPでは扱わない。

---

## 5. MVPでやらないこと

MVPでは、以下は対象外とする。

```text
複数団管理
団ごとの所属管理
詳細な権限管理
管理者による他団員目標の強制編集
ゲーム内データの自動取得
貢献度や素材数の自動連携
SlackやDiscordへの通知
提案の取り下げ
提案へのコメント機能
目標の承認フロー
非公開目標
公開範囲の細かい設定
ランキング機能
達成バッジ機能
実績の自動集計
```

MVPでは、全アカウントの目標は団内で確認できる前提とする。

## 6. 目標カテゴリ

目標カテゴリは以下を基本とする。

```text
古戦場
高難度
周回
育成
その他
```

## 6.1 古戦場

古戦場に関する目標。

例：

```text
個人10億を目指す
肉を8000個集める
本戦4日間で合計30億を目指す
HELL250を安定して回れるようにする
```

## 6.2 高難度

高難度クエストに関する目標。

例：

```text
ルシゼロに参加できる編成を作る
天元HLの担当属性を練習する
スパバハをソロ討伐する
```

## 6.3 周回

武器集めや素材集めの周回目標。

例：

```text
シエテ剣を5本集める
レヴァンス武器を覚醒15まで育てる
アガスティアを週20回周回する
```

## 6.4 育成

キャラ・武器・召喚石などの育成目標。

例：

```text
十天衆を最終する
賢者を最終する
終末武器を作成する
オメガ武器を作成する
```

## 6.5 その他

上記に当てはまらない団内活動に関する目標。

---

## 7. 目標状態

目標状態は以下を基本とする。

```text
未着手
進行中
達成
中止
```

## 7.1 未着手

目標は作成済みだが、まだ取り組みを開始していない状態。

## 7.2 進行中

目標に取り組んでいる状態。

## 7.3 達成

目標を達成した状態。

## 7.4 中止

何らかの理由で目標への取り組みをやめた状態。

---

## 8. 提案状態

提案状態は以下を基本とする。

```text
提案中
受け入れ済み
見送り
```

## 8.1 提案中

他団員から提案されたが、本人がまだ受け入れ・見送りを判断していない状態。

## 8.2 受け入れ済み

本人が提案を受け入れ、自分の目標として登録した状態。

## 8.3 見送り

本人が今回はその提案を採用しなかった状態。

## 10. 目標一覧画面

## 10.1 目的

団員全体の目標と進捗を確認するための画面。

現在は1つの団での運用のみを想定するため、すべてのアカウントの目標を一覧で表示する。

## 10.2 表示項目

```text
団員名
目標タイトル
カテゴリ
期限
進捗率
状態
最終更新日
```

## 10.3 表示形式

一覧カード形式またはテーブル形式で表示する。

MVPでは、実装しやすい形式を優先してよい。
ただし、スマートフォンでも見やすい表示を意識する。

## 10.4 フィルター

以下のフィルターを用意する。

```text
団員
カテゴリ
状態
期限
キーワード
```

## 10.5 キーワード検索対象

キーワード検索では、以下を対象とする。

```text
目標タイトル
目標内容
進捗メモ
団員名
```

## 11. 目標詳細画面

## 11.1 目的

目標の内容と進捗状況を詳しく確認するための画面。

## 11.2 表示項目

```text
目標タイトル
対象団員
カテゴリ
目標内容
目標値
現在値
単位
進捗率
期限
状態
メモ
最終更新日
作成日
提案者
```

提案から作成された目標の場合は、誰から提案されたかを表示する。

## 11.3 操作

自分の目標である場合、以下の操作ができる。

```text
進捗更新
目標編集
達成にする
中止にする
```

他団員の目標である場合、MVPでは閲覧のみとする。

---

## 12. 目標作成画面

## 12.1 目的

団員が自分の目標を作成するための画面。

## 12.2 入力項目

| 項目 | 必須 | 内容 |
|---|---|---|
| 目標タイトル | 必須 | 目標の短いタイトル |
| カテゴリ | 必須 | 古戦場 / 高難度 / 周回 / 育成 / その他 |
| 目標内容 | 任意 | 目標の詳細説明 |
| 目標値 | 任意 | 数値目標がある場合に入力 |
| 現在値 | 任意 | 初期進捗 |
| 単位 | 任意 | 億 / 個 / 本 / 回 / % / その他 |
| 期限 | 任意 | 目標の期限 |
| 状態 | 必須 | 未着手 / 進行中 |

## 12.3 入力例

```text
目標タイトル：火古戦場で個人10億
カテゴリ：古戦場
目標内容：本戦終了までに個人貢献度10億を目指す
目標値：10
現在値：0
単位：億
期限：火古戦場最終日
状態：未着手
```

---

## 13. 進捗更新画面

## 13.1 目的

自分の目標の進捗を更新するための画面。

## 13.2 入力項目

```text
現在値
進捗メモ
状態
```

## 13.3 進捗率

目標値と現在値が入力されている場合は、進捗率を自動計算する。

```text
進捗率 = 現在値 / 目標値 * 100
```

目標値が未入力の場合は、進捗率を自動計算しない。
この場合は、進捗メモと状態で進捗を表現する。

## 13.4 入力例

```text
現在値：3.2
進捗メモ：肉集めは順調。HELL150以降の速度次第で調整。
状態：進行中
```

---

## 14. 目標提案画面

## 14.1 目的

他団員に対して、取り組んでほしい目標を提案するための画面。

提案された目標は、すぐに本人の目標にはならない。
本人が受け入れた場合のみ、自分の目標として登録される。

## 14.2 入力項目

| 項目 | 必須 | 内容 |
|---|---|---|
| 提案先団員 | 必須 | 目標を提案する相手 |
| 目標タイトル | 必須 | 提案する目標のタイトル |
| カテゴリ | 必須 | 古戦場 / 高難度 / 周回 / 育成 / その他 |
| 目標内容 | 任意 | 目標の詳細 |
| 目標値 | 任意 | 数値目標がある場合に入力 |
| 単位 | 任意 | 億 / 個 / 本 / 回 / % / その他 |
| 期限 | 任意 | 目標の期限 |
| 提案メモ | 任意 | なぜ提案するか |

## 14.3 入力例

```text
提案先団員：Aさん
目標タイトル：天元HL用の水編成を作る
カテゴリ：高難度
目標内容：団内天元に参加できるよう、水属性の担当編成を準備する
期限：今月末
提案メモ：水担当が増えると団内で開催しやすくなるため
```

## 15. 提案一覧画面

## 15.1 目的

自分に届いた目標提案を確認し、受け入れるか見送るかを判断するための画面。

## 15.2 表示項目

```text
提案者
目標タイトル
カテゴリ
期限
提案メモ
提案状態
作成日
```

## 15.3 操作

提案中の目標に対して、以下の操作ができる。

```text
受け入れる
見送る
```

## 15.4 受け入れ時の挙動

提案を受け入れた場合、提案内容を元に自分の目標を作成する。

その際、以下の情報を引き継ぐ。

```text
目標タイトル
カテゴリ
目標内容
目標値
単位
期限
提案者情報
```

受け入れ後、提案状態は「受け入れ済み」にする。

## 15.5 見送り時の挙動

提案を見送った場合、提案状態は「見送り」にする。

見送り理由の入力は、MVPでは任意または対象外としてよい。

## 18. 操作フロー

## 18.1 自分で目標を作成する流れ

```text
1. 目標一覧画面を開く
2. 目標作成画面を開く
3. 目標タイトル・カテゴリ・内容・期限などを入力する
4. 保存する
5. 目標一覧に自分の目標が表示される
```

## 18.2 自分の進捗を更新する流れ

```text
1. 目標一覧または詳細画面から自分の目標を開く
2. 進捗更新画面を開く
3. 現在値・進捗メモ・状態を入力する
4. 保存する
5. 目標一覧と詳細画面に最新進捗が反映される
```

## 18.3 他団員に目標を提案する流れ

```text
1. 目標提案画面を開く
2. 提案先団員を選択する
3. 目標内容を入力する
4. 提案を保存する
5. 提案先団員の提案一覧に表示される
```

## 18.4 提案を受け入れる流れ

```text
1. 提案一覧画面を開く
2. 提案内容を確認する
3. 受け入れるを選択する
4. 提案内容を元に自分の目標が作成される
5. 提案状態が受け入れ済みに変わる
```

## 18.5 提案を見送る流れ

```text
1. 提案一覧画面を開く
2. 提案内容を確認する
3. 見送るを選択する
4. 提案状態が見送りに変わる
```

## 19. フィルター・検索要件

目標一覧画面では、以下の条件で絞り込みできるようにする。

| 条件 | 内容 |
|---|---|
| 団員 | 特定の団員の目標のみ表示 |
| カテゴリ | 古戦場 / 高難度 / 周回 / 育成 / その他 |
| 状態 | 未着手 / 進行中 / 達成 / 中止 |
| 期限 | 期限あり / 期限なし / 期限切れ |
| キーワード | 目標タイトル・目標内容・メモ・団員名を検索 |

MVPでは、すべてのフィルターを一度に高機能に作り込む必要はない。
ただし、団員・カテゴリ・状態のフィルターは優先度が高い。

## 20. 実装時の注意点

## 20.1 ノルマ管理に寄りすぎない

この機能は、団員を管理・監視するための機能ではなく、団内で目標や進捗を共有しやすくするための機能とする。

画面上の表現も、圧が強くなりすぎないようにする。

避けたい表現：

```text
未達成者
遅れている団員
ノルマ未達
```

使いやすい表現：

```text
進行中
もう少し
達成済み
目標調整中
```

## 20.2 提案は本人の承認を必須にする

他団員からの提案は、本人が受け入れるまで個人目標にはしない。

勝手に他人の目標として登録できないようにする。

## 20.3 MVPでは権限を複雑にしない

現在は1つの団での運用を前提とする。
そのため、MVPでは複数団対応や詳細な権限管理を実装しない。

ただし、将来的な拡張を考えて、データ項目には user_id を明確に持たせる。

## 20.4 数値目標がない目標も扱えるようにする

すべての目標が数値で表現できるとは限らない。

例：

```text
ルシゼロ闇編成を完成させる
天元HLの動きを覚える
古戦場前にフルオート編成を確認する
```

そのため、目標値・現在値・単位は任意入力とする。
数値目標がない場合は、状態とメモで進捗を表現する。

Codexに実装を依頼する場合は、まず以下の既存資料を確認させる。

```text
docs/01_overall_policy.md

```

そのうえで、本要件定義書に沿って、団内目標共有機能の設計・実装を行う。

実装時は、最初から高機能に作り込みすぎず、以下の順で進める。

```text
1. 目標データの表示・作成・更新
2. 目標一覧とフィルター
3. 目標詳細
4. 目標提案
5. 提案の受け入れ・見送り
6. UI調整
```

## 23. 最重要方針

この機能で最も重視することは以下。

```text
団員が自分の目標を設定できること
団内で各団員の目標と進捗を確認できること
他団員から目標を提案できること
提案は本人が受け入れた場合のみ個人目標になること
古戦場専用ではなく汎用的な団内目標共有機能として作ること
MVPでは複雑な権限や自動連携を入れず、手入力で運用できること
```

以上を、saisoku の団内目標共有機能における要件定義とする。
