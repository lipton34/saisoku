# 必要数の進捗確認機能 仕様書

## 1. 概要

### 1.1 機能名

必要数の進捗確認機能

### 1.2 目的

グランブルーファンタジーにおける武器強化・キャラクター育成について、目標達成までに必要な素材と付随条件をまとめて確認できるようにする。

ユーザーが素材と必要数を一から登録する方式ではなく、あらかじめ用意されたプリセットから目標を選択し、現在の所持数や達成状況だけを入力する方式とする。

### 1.3 本機能の価値

- 目標を選ぶだけで必要素材、育成段階、前提条件を表示できる
- 素材不足だけでなく、クエスト、Rank、キャラクターLvなどの未達条件も確認できる
- 目標達成までのどの段階で止まっているかを確認できる
- スマートフォンから所持数や達成状況を短時間で更新できる

### 1.4 本家機能との差別化

単純な素材数の登録・集計ではなく、プリセットを基に以下を一体で管理する。

- 目標達成までの段階
- 各段階で必要な素材
- 素材以外の前提条件
- 現在の達成状況
- 次に不足している素材・条件

## 2. 初期版の対象範囲

### 2.1 対象プリセット

| プリセット           | ゴール                                               |
| -------------------- | ---------------------------------------------------- |
| 終末武器             | プリセットで定義した最終強化段階まで                 |
| 十天衆               | 加入からLv150まで                                    |
| 賢者                 | 最終上限解放および4アビ取得まで                      |
| ドラゴニックウェポン | オリジン化まで                                       |
| 破壊武器             | 5凸まで                                              |
| オリジン系ジョブ     | 対象ジョブの取得およびプリセットで定義した到達点まで |
| 極星器               | 5凸および覚醒Lv最大まで                              |
| レヴァンス武器       | 4凸および覚醒Lv20まで                                |

プリセットごとに、対象キャラクター、対象武器、属性などの必要な選択肢を定義する。

上記8種類について、実装時点で必要素材・必要数・付随条件を確認できた対象は、初期データとしてコードに同梱する。未確認または正確性を保証できない対象を空のプリセットとして公開してはならない。

極星器とレヴァンス武器は、上限解放と覚醒を別の段階として管理する。武器ごとに選択可能な覚醒タイプや必要素材が異なる場合は、対象武器・覚醒タイプに応じた要件をプリセット側で定義する。

オリジン系ジョブは対象ジョブを選択できるようにし、取得までに必要な素材だけでなく、前提ジョブ、ジョブLv、クエストクリアなどの解放条件も付随条件として表示する。取得後に固有の強化段階が存在する場合は、その段階を同じプリセット内に定義できる構造とする。

### 2.2 初期版に含める機能

- プリセットから目標を登録する
- 対象キャラクター・武器・属性などを選択する
- ゴールまでの段階を自動表示する
- 段階ごとの必要素材と必要数を表示する
- 素材の所持数を登録・更新する
- 不足数と素材進捗率を自動計算する
- 素材以外の条件を表示する
- チェック式、数値入力式の条件を更新する
- 段階別および目標全体の進捗を表示する
- 不足素材と未達成条件を優先表示する
- 複数の目標を登録・削除する
- スマートフォン向けの表示・操作に対応する

### 2.3 初期版に含めない機能

- ユーザーによる自由な素材・必要数の登録
- 複数目標の必要素材の合算
- 複数目標間での所持素材の割り当て
- 目標の優先順位設定
- ゲームアカウントとの自動連携
- スクリーンショットからの所持数読み取り
- 周回数、必要日数、1日あたり目標数の計算
- ユーザーによる独自プリセット作成
- ユーザーによる既存プリセットの編集
- プリセットを追加・編集する管理画面
- 素材消費後の所持数自動減算

## 3. 用語

| 用語         | 定義                                                               |
| ------------ | ------------------------------------------------------------------ |
| プリセット   | 目標、対象候補、段階、必要素材、付随条件をあらかじめ定義したデータ |
| ユーザー目標 | ユーザーがプリセットから登録した進捗管理対象                       |
| 段階         | 加入、最終上限解放、Lv110、Lv120など、目標達成までの区切り         |
| 素材要件     | アイテム名と必要数で構成される条件                                 |
| 付随条件     | Rank、キャラクターLv、クエストクリア、武器強化など、素材以外の条件 |
| 所持数       | ユーザーが現在所持している素材数                                   |
| 不足数       | 必要数に対して不足している素材数                                   |
| 達成済み     | 素材要件と付随条件をすべて満たした状態                             |

## 4. 基本的な利用フロー

### 4.1 目標の登録

1. ユーザーが「目標を追加」を選択する
2. プリセットを選択する
3. 対象キャラクター、武器、属性などを選択する
4. 必要に応じてゴールを選択する
5. 現在到達している段階を選択する
6. 内容を確認して目標を登録する
7. ゴールまでの段階、必要素材、付随条件が自動生成される

### 4.2 進捗の更新

1. ユーザーが目標詳細を開く
2. 未達成の段階を展開する
3. 素材の所持数を入力する
4. クエストクリアなどの条件をチェックする
5. RankやLvなどの現在値を入力する
6. 不足数、進捗率、段階の達成状態が即時更新される

### 4.3 進捗の確認

1. 目標一覧で全体の達成状況を確認する
2. 目標詳細で現在の段階を確認する
3. 不足素材と未達成条件を確認する
4. 達成済み段階は必要に応じて展開して確認する

## 5. 画面仕様

### 5.1 目標一覧画面

#### 表示項目

- 目標名
- 対象キャラクター・武器名
- ゴール
- 現在の段階
- 完了段階数／全段階数
- 全体プログレスバー
- 次に未達成となっている段階
- 不足素材の種類数
- 未達成条件数

#### 操作

- 目標詳細を開く
- 目標を追加する
- 目標を削除する

#### 表示例

```text
シエテ Lv150まで
進捗 5 / 8段階
[████████░░░░] 63%

現在：Lv130への限界超越
不足素材 4種類・未達成条件 1件
```

### 5.2 目標登録画面

#### 入力項目

| 項目       |     必須 | 内容                                   |
| ---------- | -------: | -------------------------------------- |
| プリセット |     必須 | 初期プリセットから選択                 |
| 対象       |     必須 | キャラクター、武器などを選択           |
| 属性・種類 | 条件付き | プリセットで必要な場合のみ表示         |
| ゴール     |     必須 | プリセットで選択可能な到達点を表示     |
| 現在の段階 |     必須 | 登録時点ですでに達成している段階を指定 |

#### 現在の段階の扱い

- 選択した段階より前の段階を達成済みとして登録する
- 選択した段階自体を達成済みに含めるか、文言で明示する
- 推奨文言は「達成済みの最終段階」とする
- 「未着手」を選択可能にする

### 5.3 目標詳細画面

#### 画面上部

- 目標名
- 対象名
- ゴール
- 完了段階数／全段階数
- 全体プログレスバー
- 目標削除メニュー

#### 段階一覧

各段階をアコーディオン形式で表示する。

- 達成済み段階：初期状態では閉じる
- 現在取り組む段階：初期状態で開く
- 将来の段階：初期状態では閉じる
- ユーザーは任意の段階を開閉できる

#### 段階内の表示順

1. 未達成の付随条件
2. 不足している素材
3. 達成済みの付随条件
4. 必要数を満たした素材

達成済み項目はまとめて折りたたみ可能にする。

#### 素材行

- 素材名
- 必要数
- 所持数入力欄
- 不足数
- 素材ごとのプログレスバーまたは進捗率
- 達成状態

#### 付随条件行

- 条件名
- 条件種別に応じた入力欄
- 条件の補足説明
- 達成状態

#### 表示例

```text
シエテ Lv150まで
全体進捗 5 / 8段階

✓ 加入
✓ 最終上限解放
✓ Lv110
✓ Lv120
▼ Lv130
  条件 2 / 3完了
  素材 8 / 12種類達成

  未達成の条件
  □ 対象のフェイトエピソードをクリア

  不足素材
  素材A  50 / 100  あと50
  素材B   2 / 10   あと8

▶ Lv140
▶ Lv150
```

## 6. 付随条件の仕様

### 6.1 条件種別

| 種別                 | 表示例                     | 入力方法           | 達成判定           |
| -------------------- | -------------------------- | ------------------ | ------------------ |
| チェック             | 指定クエストをクリア       | チェックボックス   | チェック済み       |
| Rank                 | Rank300以上                | 現在Rankを数値入力 | 現在値が必要値以上 |
| キャラクターLv       | シエテLv120以上            | 現在Lvを数値入力   | 現在値が必要値以上 |
| 武器強化             | 対象武器を5凸              | チェックボックス   | チェック済み       |
| キャラクター条件     | 十天衆全員が条件を達成     | チェックボックス   | チェック済み       |
| クエスト・エピソード | フェイトエピソードをクリア | チェックボックス   | チェック済み       |
| 数値条件             | 必要ルピを用意             | 現在値を数値入力   | 現在値が必要値以上 |
| 説明のみ             | 特定の手順を実施           | 確認チェック       | チェック済み       |

### 6.2 条件の補足

条件には必要に応じて次の情報を設定できる。

- 条件を達成する場所・方法
- 前提となる別の段階
- 対象クエスト名
- 注意事項
- 条件が必要となる理由

補足は常時表示せず、詳細表示または情報アイコンから確認できる形式を基本とする。

### 6.3 共通値

同じユーザーRankなど、複数目標で共通利用できる値はユーザー単位で保持する。ある目標でRankを更新した場合、ほかの目標のRank条件にも反映する。

キャラクター固有Lvや対象武器の状態は、対象単位で保持する。

## 7. 素材所持数の仕様

### 7.1 共通管理

素材所持数は目標ごとではなく、ユーザーと素材の組み合わせで一つ保持する。

同じ素材が複数の目標に含まれる場合、一つの画面で所持数を更新すると、ほかの目標にも同じ値を表示する。

### 7.2 初期版での判定

各目標は独立して所持数を判定する。

例：素材Aの所持数が60個で、目標Xに100個、目標Yに50個必要な場合

- 目標X：60／100、不足40
- 目標Y：60／50、不足0

初期版では、合計150個に対して不足90個とは判定しない。また、目標Yに50個を割り当てて目標Xの利用可能数を10個に減らす処理も行わない。

### 7.3 入力制約

- 0以上の整数のみ入力可能
- 空欄は0として扱う
- マイナス値、小数、数値以外は保存しない
- 極端に大きい値にはシステム上の上限を設定する
- 入力変更後、保存と再計算を行う

### 7.4 更新方式

スマートフォンで素早く更新できるよう、詳細な編集画面へ遷移せず、素材行の数値欄から直接変更できるようにする。

連続入力を妨げないため、保存は次のいずれかとする。

- 入力欄からフォーカスが外れたときに保存
- 入力停止後、短い待機時間を置いて自動保存

保存中、保存完了、保存失敗の状態を表示する。

## 8. 進捗計算・達成判定

### 8.1 素材の不足数

```text
不足数 = max(必要数 - 所持数, 0)
```

### 8.2 素材ごとの進捗率

```text
素材進捗率 = min(所持数 / 必要数 × 100, 100)
```

必要数が0のデータはプリセット不備として扱い、計算対象に含めない。

### 8.3 素材要件の達成

```text
所持数 >= 必要数
```

### 8.4 段階の達成

原則として、次の両方を満たした場合に達成とする。

- 段階内の全素材要件が達成済み
- 段階内の全付随条件が達成済み

ただし、登録時に「達成済みの最終段階」として指定された段階以前は、過去の素材消費を考慮し、個別の所持数にかかわらず達成済みとして扱う。

### 8.5 段階の手動完了

素材を実際に消費すると所持数が減るため、完了済み段階を現在の所持数だけで再判定して未達成へ戻してはならない。

そのため、段階には以下の状態を保持する。

- 自動判定中
- 手動で達成済み

自動判定中の段階は、全要件を満たすと達成候補として表示する。ユーザーがゲーム内で実際に強化・解放を終えたことを確認し、「この段階を完了」にすることで手動達成済みとなる。

### 8.6 全体進捗

初期版では、段階の完了数を基準に計算する。

```text
全体進捗率 = 完了段階数 / 全段階数 × 100
```

素材数の単純平均は使用しない。

### 8.7 現在の段階

段階順の先頭から確認し、最初に未完了となっている段階を「現在の段階」とする。

後続段階の素材や条件も閲覧・更新可能とする。前の段階が未完了でも、後続段階の準備を妨げない。

## 9. プリセット仕様

### 9.1 プリセットの構成要素

各プリセットは次の情報を持つ。

- プリセットID
- プリセット名
- 説明
- 対象種別
- 対象候補
- 選択可能なゴール
- 段階一覧
- 段階の表示順
- 段階ごとの素材要件
- 段階ごとの付随条件
- 補足説明
- データ版数
- 有効／無効状態

### 9.2 段階の構成要素

- 段階ID
- 段階名
- 表示順
- 必要素材一覧
- 付随条件一覧
- 前提段階
- 補足説明

### 9.3 プリセット更新

ゲーム側の仕様変更や誤りの修正に対応できるよう、プリセットデータは画面ロジックから分離して管理する。

プリセットの追加・変更・削除は、管理者がコード上の定義を変更してリリースする。利用者向けの追加・編集機能、および管理画面からの直接編集機能は設けない。

新しいプリセットや対象の追加要望は、管理者が内容を確認し、必要素材・必要数・付随条件の正確性を検証したうえで採否を決定する。採用した場合は、通常のコード変更と同様にレビュー・テストを経て反映する。

既存のユーザー目標に対する更新方針は次のとおりとする。

- 表記修正・説明追加：既存目標にも反映する
- 必要素材・必要条件の変更：変更内容をユーザーに通知して反映する
- 段階構造の大幅変更：既存目標を自動更新せず、移行確認を行う

初期実装では、プリセット更新時の自動移行を必須としない。ただし、将来対応できるようデータ版数を保持する。

### 9.4 プリセットデータの調査

各プリセットの具体的な必要素材、必要数、付随条件は、実装前にゲーム内情報または信頼できる最新情報を基に別途確定する。

確認できたデータは初期実装時に投入する。調査が間に合わない、情報が競合している、または正確性を確認できないデータは公開対象から外し、確認後に管理者がコードから追加する。

調査時には次を明確にする。

- 対象ごとの差分
- 属性ごとの差分
- 選択ルートによる差分
- 過去段階からの累計か、各段階単独の必要数か
- 条件の適用時期
- 最新アップデートによる変更有無

## 10. データ構造案

### 10.1 主なデータ

| データ                | 主な内容                                     |
| --------------------- | -------------------------------------------- |
| Item                  | 素材ID、素材名、表示情報                     |
| Preset                | プリセット名、対象種別、版数                 |
| PresetTarget          | 選択可能なキャラクター・武器・属性           |
| PresetStage           | 段階名、表示順、前提段階                     |
| StageItemRequirement  | 段階、素材、必要数                           |
| StageCondition        | 条件名、条件種別、必要値、補足               |
| UserGoal              | ユーザー、プリセット、対象、ゴール、使用版数 |
| UserItemInventory     | ユーザー、素材、所持数                       |
| UserConditionProgress | ユーザー目標、条件、現在値またはチェック状態 |
| UserStageProgress     | ユーザー目標、段階、完了状態、完了日時       |
| UserSharedValue       | ユーザーRankなどの共通値                     |

### 10.2 関係

- 一つのプリセットは複数の段階を持つ
- 一つの段階は複数の素材要件と付随条件を持つ
- 一人のユーザーは複数のユーザー目標を持つ
- 一つの素材所持数は同一ユーザーの全目標から参照される
- 段階の完了状態はユーザー目標ごとに保持する

## 11. 操作・表示ルール

### 11.1 スマートフォン優先

- 1列表示を基本とする
- 重要情報を横スクロールさせない
- 数値入力欄はタップしやすい大きさにする
- 数値入力時は数字キーボードを表示する
- 保存ボタンを何度も押させない
- 未達成項目を画面上部に表示する
- 長い段階一覧はアコーディオンで整理する
- 色だけに依存せず、アイコンや文言でも達成状態を示す

### 11.2 状態表示

| 状態     | 表示方針                                         |
| -------- | ------------------------------------------------ |
| 達成済み | チェックアイコンと「達成済み」                   |
| 未達成   | 不足数または未達成条件を表示                     |
| 達成可能 | 全要件を満たし、ゲーム内操作待ちであることを表示 |
| 保存中   | 小さなローディング表示                           |
| 保存失敗 | 入力値を残したまま再試行可能にする               |

### 11.3 削除確認

目標削除時は確認ダイアログを表示する。

目標を削除しても、共通管理している素材所持数は削除しない。

---

## 12. バリデーション・例外処理

- 必須項目が未選択の場合は目標を登録できない
- 同じプリセット・対象・ゴールの重複登録時は警告する
- 素材所持数に不正な値を入力した場合は保存せず、理由を表示する
- 通信失敗時は入力内容を保持し、再試行できるようにする
- プリセットが無効化された既存目標は閲覧可能とし、警告を表示する
- プリセットデータに不足や矛盾がある場合、誤った達成判定をせずエラー状態として表示する
- 削除された素材や条件を参照する目標がある場合、履歴を失わないよう論理削除を使用する

---

## 13. 非機能要件

### 13.1 性能

- 目標一覧は通常の利用件数で待ち時間を感じにくい速度で表示する
- 所持数の変更後、画面上の不足数と進捗率は即時に再計算する
- 保存処理中も別項目の入力を妨げない

### 13.2 保守性

- プリセットデータと画面処理を分離する
- プリセットはコード上の静的データまたはシードデータとして管理する
- プリセット変更はコードレビューとテストを経て反映できるようにする
- 素材や条件の追加で画面コードの修正を最小限にする
- プリセットに版数を持たせる
- 条件種別を追加できる構造にする

### 13.3 データ整合性

- 同一ユーザー・同一素材の所持数を重複保持しない
- 段階完了後に素材所持数が減っても完了状態を維持する
- 複数端末から更新した場合は、最後に正常保存された値を採用する

---

## 14. 受け入れ条件

### 14.1 目標登録

- 8種類の初期プリセットが選択できる
- 初期実装時に確認済みのプリセットデータが投入されている
- プリセットに応じた対象とゴールが選択できる
- 登録後、ゴールまでの段階が正しい順序で表示される
- 現在段階より前を達成済みとして登録できる
- 極星器では5凸と覚醒Lv最大までの進捗を確認できる
- レヴァンス武器では4凸と覚醒Lv20までの進捗を確認できる
- オリジン系ジョブでは対象ジョブの素材要件と解放条件を確認できる

### 14.2 素材進捗

- 所持数を入力すると不足数と進捗率が再計算される
- 必要数以上を所持している場合、不足数は0、進捗率は100%となる
- 同じ素材の所持数が別目標にも反映される
- 複数目標の必要数は合算されない

### 14.3 条件進捗

- チェック条件を更新できる
- Rank・Lv・数値条件を入力できる
- 必要値以上の場合に条件が達成済みとなる
- 共通Rankの更新がほかの目標にも反映される

### 14.4 段階・全体進捗

- 段階内の全要件を満たすと「達成可能」と表示される
- ユーザーが段階完了を確定できる
- 完了後に所持素材が減っても段階が未完了へ戻らない
- 完了段階数を基に全体進捗率が表示される
- 最初の未完了段階が現在の段階として表示される

### 14.5 スマートフォン表示

- 横スクロールなしで主要情報を確認できる
- 段階を開閉できる
- 素材所持数を詳細編集画面へ移動せず変更できる
- 未達成条件と不足素材が達成済み項目より先に表示される

## 15. 実装前に確定する事項

1. 各プリセットの正式名称と対象範囲
2. 各プリセットの対象キャラクター・武器・属性一覧
3. 各段階の名称と順序
4. 各段階の正確な必要素材・必要数
5. 各段階の正確な付随条件
6. 素材マスターの既存データを流用できるか
7. 既存のユーザー認証・保存方式との接続方法
8. 同一目標の重複登録を許可するか
9. 自動保存の具体的なタイミング
10. プリセット更新時の既存目標への反映方法
11. オリジン系ジョブに含める対象ジョブと正式なゴール
12. 極星器の覚醒Lv最大値および対象武器・覚醒タイプごとの差分
13. レヴァンス武器の対象武器・覚醒タイプごとの差分

プリセットの追加・編集をユーザー操作で行わず、管理者によるコード変更のみとする方針は確定済みとする。

## 17. 初期版の完成イメージ

プリセットと対象を選ぶだけでゴールまでの段階が作成され、ユーザーは素材所持数と付随条件だけを更新する。画面を開けば、現在の段階、足りない素材、未達成の条件、ゴールまでの進捗をスマートフォン上で短時間に確認できる状態を初期版の完成とする。
