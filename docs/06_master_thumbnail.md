# Codex作業指示：06_キャラ・召喚石・武器マスタとサムネイル

## 1. 目的

`saisoku` に、高難易度向け編成共有機能で利用する **キャラ・召喚石・武器・主人公ジョブの候補マスタ** と、**小型サムネイル画像の管理方針** を追加・整備する。

編成投稿では画像投稿中心ではなく、候補リストからキャラ・召喚石・武器を選択する方式を基本とする。

このため、選択候補として使うマスタと、補助表示用の小型サムネイルを用意する。

## 3. 基本方針

```text
最初から全件管理しない
高難度攻略用・レヴァンス周回用で使うものを優先する
候補にないものは自由入力できる
自由入力されたものは後から候補化できる
画像は小型サムネイルのみ自前保持する
Supabase Freeで収まる範囲に抑える
```

重要なのは、**マスタが未完成でも編成投稿できること**である。

```text
候補リストにあるものは選択
候補にないものは自由入力
必要になったら後から候補に追加
```

## 5. キャラマスタの対象範囲

キャラは編成共有で最も検索価値が高いため、優先度を高くする。

初期対象：

```text
全リミテッドキャラ
最終上限解放実装済みキャラ
十天衆
十賢者
十二神将
全季節限定キャラ
高難易度・レヴァンス周回でよく使われる恒常キャラ
```

MVPで無理に対象にしないもの：

```text
全Rキャラ
全SRキャラ
古い編成でしか使わないキャラ
高難度・周回用途と関係が薄いキャラ
```

ただし、SRやRでも特定用途で必要なら、自由入力または個別追加できるようにする。

### キャラ候補に持たせたい情報

| 項目 | 内容 |
|---|---|
| キャラ名 | 日本語名を基本 |
| 属性 | 火・水・土・風・光・闇 |
| レアリティ | SSR / SR / R |
| 種別 | キャラ |
| サムネイル | 小型画像 |
| 別バージョン識別 | 水着、浴衣、リミテッドなど |
| 参照URL | 任意。GBF Wikiなど |
| メモ | 団内用補足 |

## 9. サムネイル画像の扱い

画像は外部URL参照ではなく、saisoku側で自前保持する。

方針：

```text
画像は小型サムネイルのみ
高解像度画像は持たない
編成スクショの大量保存はしない
外部画像URLに依存しない
Supabase Storageに保存する
```

対象：

```text
キャラサムネイル
召喚石サムネイル
武器サムネイル
```

画像サイズ目安：

```text
1枚30KB〜80KB
目標は50KB前後
初期保存容量は300MB以内
```

Supabase Free運用を前提に、一覧で大量画像を一度に表示しないことも考慮する。

---

## 10. 初期画像総数の想定

初期対象の画像総数は、以下を目安とする。

```text
キャラ：400〜550枚
召喚石：70〜100枚
武器：350〜450枚
合計：800〜1,200枚程度
```

1枚50KB前後にする場合、初期ストレージ使用量は以下を想定する。

```text
800枚：約40MB
1,000枚：約50MB
1,200枚：約60MB
余裕を見ても100MB前後
```

Supabase FreeのStorage範囲内で十分運用可能と判断する。

## 11. 初期登録範囲の進め方

段階的に増やす。

### フェーズ1：最低限

```text
ルシゼロ・天元HL・スパバハで使う主要キャラ
レヴァンス系で使う主要キャラ
主要召喚石
主要ジョブ
重要武器のみ
```

### フェーズ2：団内運用で追加

```text
団員が実際に投稿した編成で使われたもの
自由入力されたもののうち、再利用されそうなもの
団内で不足確認したいキャラ・石・武器
```

### フェーズ3：範囲拡張

```text
属性ごとの主要キャラ
神石/マグナ別の主要武器
新コンテンツで使われるもの
環境変化で使われるようになったもの
```

## 12. 自由入力から候補化する方針

候補にないものが出てきた場合、投稿者が自由入力できるようにする。

そのうえで、後から管理者が候補として正式登録できるようにする。

```text
自由入力
↓
投稿にはそのまま使える
↓
よく使われるものを管理者が候補化
↓
次回以降は選択式で使える
```

これにより、マスタ未整備でも運用を止めずに済む。

---

## 13. 日本語名優先

候補リストでは日本語名を基本とする。

理由は、団内利用では日本語名で検索する可能性が高いため。

MVP：

```text
日本語名
略称・通称はメモまたはタグで対応
```

将来検討：

```text
英語名
別名
略称
季節限定識別
シリーズ識別
```

## 14. マスタ・画像準備の半自動化

キャラ・召喚石・武器マスタの初期作成は、完全手作業ではなく半自動化する。

GBF Wiki / GBF Asset Lookup などの外部公開情報を参考に、以下を収集する。

```text
名称
属性
カテゴリ
参照URL
画像候補
```

ただし、saisoku本体の通常機能として外部サイトを常時スクレイピングしない。

初期データ準備用の補助スクリプトとして扱い、取得後は人間が確認してから取り込む。

画像は小型サムネイル化して、saisoku側で自前保持する。

## 17. 完了条件

以下ができれば、この機能のMVPは完了とする。

```text
編成投稿時にキャラ・召喚石・武器・ジョブを候補から選択できる
候補にないものは自由入力できる
小型サムネイルを表示できる
候補マスタは高難度・レヴァンスで使うものを中心に扱える
自由入力されたものを後から候補化できる運用にできる
画像はSupabase Freeで運用できる範囲に抑えられる
```
