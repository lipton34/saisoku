# 今回の主な修正目的

今回の目的は、投稿フォームを「単なる縦長フォーム」ではなく、グラブルの編成画面を参考にした、見やすく操作しやすいUIへ修正することです。

ただし、グラブル本家UIの完全再現は不要です。  
参考にするべきなのは、見た目の細部ではなく、以下の情報構造と操作感です。

- キャラ・武器・召喚石がそれぞれ枠として並んでいる
- 現在選択中の枠が分かる
- 候補一覧から選ぶと、選択中の枠へ反映される
- 選択済みの枠をクリックして差し替えできる
- サムネイル画像を中心に、名前・属性・種別などを補助表示する
- 武器や召喚石はグリッド状に並ぶ
- 入力フォームというより「編成を組む画面」に見える

# 1. `content-grid` の横幅制約を見直す

現在のHTML構造は以下のようになっています。

```html
<section class="content-grid">
  <section class="build-form-steps-container">
    <form class="build-form-steps">
      ...
    </form>
  </section>
</section>
```

現状では `content-grid` の max-width が狭く、投稿フォーム全体が窮屈に見えます。  
そのため、投稿フォーム画面ではより広い横幅を使えるようにしてください。

## 修正方針

`content-grid` の max-width を広げてください。

ただし、`content-grid` が他画面でも使われている場合、全画面に影響が出ないようにしてください。

推奨方針は以下です。

### 推奨案

投稿フォーム画面専用の modifier class または親classを追加してください。

例：

```html
<section class="content-grid content-grid--build-form">
  <section class="build-form-steps-container">
    ...
  </section>
</section>
```

または、

```html
<main class="build-form-page">
  <section class="content-grid">
    ...
  </section>
</main>
```

そのうえで、投稿フォーム画面だけ横幅を広げてください。

例：

```css
.content-grid--build-form {
  max-width: 1280px;
  width: min(100%, 1280px);
}
```

または、

```css
.build-form-page .content-grid {
  max-width: 1280px;
  width: min(100%, 1280px);
}
```

## 注意点

- `content-grid` をグローバルに変更して、他画面のレイアウトを壊さないこと
- 一覧画面・詳細画面・ログイン画面などに影響しないようにすること
- 投稿フォーム画面のみ広げること
- PCでは 1200px〜1440px 程度まで使えるようにしてよい
- スマホでは従来どおり横スクロールが出ないようにすること

# 3. 「グラブル本家風UI」の解釈を明確化する

前回の「グラブル本家の編成画面に近いUI」という指示が十分に伝わっていなかったため、以下のように解釈して実装してください。

## グラブル本家風として参考にする点

完全な見た目のコピーではなく、以下のUI構造を参考にしてください。

### キャラ編成

- 主人公枠がある
- フロントメンバー枠が横並びで表示される
- サブメンバー枠が別グループで表示される
- 各枠はカードまたはスロットとして表示する
- 未選択枠は「＋」「未選択」などで表示する
- 選択済み枠にはサムネイル、名前、属性、役割などを表示する
- 現在編集中の枠は枠線・背景・ラベルで強調する

### 武器編成

- 武器はグリッド状に並べる
- メイン武器枠が分かる場合は明示する
- その他武器枠は複数のスロットとして表示する
- 10枠を基本にしつつ、必要であれば追加できる
- 各スロットにはサムネイル、武器名、重要度、本数を表示する
- 未選択枠は「＋」または空スロットとして表示する
- 候補一覧から選ぶと、現在選択中の武器枠に反映する

### 召喚石編成

- メイン召喚石枠を分かりやすく表示する
- フレンド召喚石候補を分かりやすく表示する
- サブ召喚石枠をグリッド状に表示する
- サブ加護枠はMVPでは必須ではないが、必要なら任意枠として扱う
- 各スロットにはサムネイル、召喚石名、属性、重要度を表示する
- 未選択枠は「＋」または空スロットとして表示する

## 実装上の表現

以下のようなクラス・構造を追加しても構いません。

formation-slot
formation-slot--active

# 7. Step 5：その他情報入力のレイアウト修正

現在確認できている Step 5 のHTML構成では、以下のような構造になっています。

```html
<div class="selected-part-picker">
  <div class="selected-part-chip-grid">
    <div class="selected-part-chip">
      <span>ハーゼリーラ</span>
      <button type="button">必須</button>
      <button type="button">推奨</button>
      <button type="button">代用可</button>
    </div>
  </div>
</div>

<div class="form-row">
  <label>必須パーツ<textarea rows="3"></textarea></label>
  <label>推奨パーツ<textarea rows="3"></textarea></label>
</div>

<div class="reference-list">
  <div class="reference-item">
    <select></select>
    <input placeholder="タイトル">
    <input placeholder="https://..." type="url">
    <input placeholder="メモ">
    <button class="icon-button danger" type="button"></button>
  </div>
</div>
```

この Step 5 は、横幅不足の影響でかなり窮屈に見える可能性があります。  
`content-grid` の幅を広げたうえで、Step 5 自体も見やすくしてください。

## 修正方針

### 選択済みパーツ

`selected-part-picker` は、単なるチップ一覧ではなく、Step 2〜4で選んだパーツの一覧として見やすくしてください。

- チップが横に詰まりすぎないようにする
- 長い名前でも崩れないようにする
- 「必須」「推奨」「代用可」ボタンが窮屈に見えないようにする
- スマホではチップを1列または自然な折り返しにする
- ボタンは小さすぎないようにする

推奨CSS方針：

```css
.selected-part-chip-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap: 12px;
}

.selected-part-chip {
  min-width: 0;
}
```

### textarea群

現在の `form-row` は、横幅が狭い場合に見づらくなります。

- PCでは2カラムでよい
- 狭い画面では1カラムにする
- textarea の横幅が潰れないようにする
- label の中身が読みやすいようにする

推奨CSS方針：

```css
.form-row {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 16px;
}
```

### 参考URL

`reference-item` は select / title / url / memo / delete button が横並びになっているため、狭い幅では崩れやすいです。

以下のようにしてください。

- PCでは横並びまたはグリッドでよい
- 中幅以下では2列または1列に落とす
- URL欄だけが極端に狭くならないようにする
- 削除ボタンが見切れないようにする
- スマホでは1項目1カードのように見せる

推奨CSS方針：

```css
.reference-item {
  display: grid;
  grid-template-columns: minmax(120px, 160px) minmax(160px, 1fr) minmax(220px, 1.5fr) minmax(160px, 1fr) auto;
  gap: 10px;
  align-items: center;
}

@media (max-width: 900px) {
  .reference-item {
    grid-template-columns: 1fr 1fr;
  }
}

@media (max-width: 640px) {
  .reference-item {
    grid-template-columns: 1fr;
  }
}
```

# 8. 候補一覧・検索パネルの共通UIを整える

キャラ・武器・召喚石の候補一覧は、共通したUIにしてください。

## 共通要件

- 検索欄を上部に置く
- 候補一覧はカードグリッドにする
- サムネイル、名前、属性、種別を表示する
- クリックまたはタップで現在編集中の枠に反映する
- 検索結果が0件のときは自由入力導線を表示する
- サムネイルがない場合はプレースホルダーを表示する
- 候補一覧が多すぎても画面全体が伸びすぎないようにする

## 推奨構造

```text
[candidate-panel]
  [検索欄]
  [フィルタが必要なら簡易フィルタ]
  [候補カードグリッド]
  [候補にない場合の自由入力]
```

## 推奨CSS方針

```css
.candidate-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(120px, 1fr));
  gap: 12px;
  max-height: 420px;
  overflow: auto;
}
```

スマホではカードが小さくなりすぎないようにしてください。
