# Codex指示書：/goals をカンバン中心UIへ再構成する

## 背景

事前調査の結果、現在の目標画面は以下の構成であることが分かった。

- `/goals` は `src/pages/GoalsPage.tsx` に集約されている
- `GoalsPage.tsx` 内に、タブ、一覧、目標ボード、作成、提案、届いた提案、右側詳細ペイン、カード、詳細表示がほぼ集約されている
- `/goals/:id` または `/goals/:goalId` の詳細ページは未定義
- 目標詳細は現在、`selectedGoal` state を使って右側ペインに表示している
- 右側詳細ペインの中身は `GoalDetail` だが、現在は `GoalsPage.tsx` 内のローカル関数である
- APIとしては `GET /api/shared-goals/:id` が既に存在し、`api.sharedGoal(id)` から利用できる
- カンバン列は `goalBoardStatuses` で定義されている
- 状態変更セレクトは `GoalBoardCard` 内に存在する
- 一覧タブと目標ボードタブは役割が重複している
- 右側詳細ペインがカンバンの横幅を圧迫している

今回の実装では、目標管理のメインをカンバン型の目標ボードに統一する。

---

## 今回の実装方針

以下の方針で修正する。

- `/goals` の初期表示を目標ボードにする
- 一覧タブは削除してよい
- 右側詳細ペインは完全に削除してよい
- 目標ボードを画面横幅いっぱいに表示する
- カード全体クリックで `/goals/:goalId` の詳細ページへ遷移する
- カード内の状態変更セレクトは残す
- セレクト操作時にカードクリック遷移が発火しないようにする
- `GoalDetail` を詳細ページでも使えるように切り出す
- ヘッダーとタブはコンパクトに再設計する
- 大画面・通常PC・タブレット・スマホ幅に対応する

---

## 今回の対象外

今回は以下は実装しない。

- 目標と編成の連結追加
- 目標と必要武器の連結追加
- 目標と討伐数の連結追加
- サブタスク機能の新規追加
- 編成スクショアップロード
- ドラッグ＆ドロップによるカンバン移動
- Prisma schema の変更
- DB migration の追加
- APIの大規模変更

既に存在する関連編成・必要武器・討伐目標・サブタスク表示や操作が `GoalDetail` 内にある場合は、詳細ページでも引き続き使えるようにする。

# 3. GoalDetailPage の追加

## 対象ファイル

- 新規：`src/pages/GoalDetailPage.tsx`
- `src/lib/api.ts`

## 実装内容

`/goals/:goalId` で表示する目標詳細ページを作成する。

詳細ページでは、URLパラメータの `goalId` を使って単体取得する。

既に以下が存在する。

```text
api.sharedGoal(id)
GET /api/shared-goals/:id
```

これを利用する。

## 表示内容

最低限、現在右側詳細ペインで表示していた内容を詳細ページで確認できるようにする。

表示候補：

- 目標名
- 団員
- カテゴリ
- 状態
- ボード状態
- 優先度
- 負荷
- 初心者向け
- 期限
- 進捗
- メモ
- 編集
- 削除
- 関連編成
- 必要武器
- 討伐目標
- サブタスク
- 目標ボードへ戻る導線

既に `GoalDetail` に関連リソース表示・更新処理がある場合は、そのまま流用する。

## 更新後の再取得

詳細ページでは、`loadGoals` 前提ではなく、単体取得の再読み込み関数を用意する。

例：

```ts
refreshGoal(goalId);
```

編集、進捗更新、関連リソース更新後は、詳細ページ上の対象目標を再取得できるようにする。

# 7. 目標ボードの横幅を広げる

## 対象ファイル

- `src/pages/GoalsPage.tsx`
- `src/styles.css`

## 現在の問題

現在は `.goals-layout` が以下のような2カラム構成になっている。

```text
minmax(0, 1fr) minmax(320px, 0.42fr)
```

また `.page-stack` の `max-width: 1180px` により、画面全体の幅も制限されている。

そのため、大画面でも余白が多く、カンバンが窮屈になっている。

## 修正方針

目標ボード画面では、専用の広めレイアウトを使う。

例：

```css
.goals-page-wide {
  width: min(100%, 1680px);
  margin: 0 auto;
}
```

既存クラス名に合わせて調整する。

## カンバン列

カンバンは以下の5列を横並びにする。

- 今やる
- 次にやる
- 後でやる
- 保留
- 完了

CSS方針：

```css
.goal-board {
  display: grid;
  grid-template-columns: repeat(5, minmax(240px, 1fr));
  gap: 16px;
  overflow-x: auto;
}
```

大画面では列幅を広めにする。

例：

```css
@media (min-width: 1600px) {
  .goal-board {
    grid-template-columns: repeat(5, minmax(280px, 1fr));
  }
}
```

画面幅が足りない場合は、カンバン部分だけ横スクロールにする。

例：

```css
@media (max-width: 1199px) {
  .goal-board {
    grid-template-columns: repeat(5, minmax(240px, 280px));
    overflow-x: auto;
  }
}
```

実際のクラス名・既存CSSに合わせて調整する。

# 8. 大画面・小画面対応

今回、画面サイズへのレンダリング対応も含める。

## 大画面

対象目安：

```text
1440px以上
1600px以上
1920px前後
```

方針：

- 中央に小さく固まりすぎない
- 目標ボードは横幅を有効活用する
- 5列が余裕を持って見える
- 余白が大きくなりすぎない
- カード幅が極端に広がりすぎる場合は、列幅・gapで調整する

## 通常PC幅

対象目安：

```text
1200px〜1439px
```

方針：

- 5列を横並びで表示する
- 列が狭くなりすぎないようにする
- 必要に応じて横スクロールを許可する

## タブレット幅

対象目安：

```text
768px〜1199px
```

方針：

- 5列横並びにこだわりすぎない
- カンバン部分を横スクロール可能にする
- 各列に最小幅を持たせる
- フィルターは折り返してよい

## スマホ幅

対象目安：

```text
767px以下
```

方針：

- カンバン部分は横スクロールで表示する
- 列の最小幅を確保する
- ヘッダーとタブを詰めすぎない
- カードは読みやすい幅を維持する

# 9. カードクリックで詳細ページへ遷移

## 対象ファイル

- `src/pages/GoalsPage.tsx`

## 実装内容

`GoalBoardCard` のカード全体をクリック可能にする。

クリック時は以下へ遷移する。

```text
/goals/:goalId
```

例：

```ts
navigate(`/goals/${goal.id}`);
```

## 注意

カード内の状態変更セレクトは残す。

状態変更セレクト操作時に、カードクリックの遷移が発火しないようにする。

必要に応じて以下を使う。

```ts
event.stopPropagation();
```

対象：

- 状態変更セレクト
- 編集ボタンや削除ボタンなど、カード内に操作UIがある場合
- その他クリック操作を持つ要素

## カード表示

カード内には、最低限以下を表示する。

- 目標タイトル
- カテゴリ
- 優先度
- 負荷
- 期限
- 進捗
- 初心者向けラベル
- 状態変更セレクト

詳細な情報は詳細ページで表示する。

# 最重要ポイント

今回の修正で最も重要なのは以下である。

```text
目標ボードを、初心者や復帰者が「今なにをすればいいか」を確認するためのメイン画面にする。
```

そのため、目標ボードには詳細情報を詰め込みすぎない。

カンバンでは状態ごとの見通しを優先し、詳細情報は `/goals/:goalId` に分離する。
