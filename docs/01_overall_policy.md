# saisoku 全体方針書 v0.1

## 1. この文書の目的

この文書は、saisoku の機能開発を Codex に依頼する際、各機能別指示書の前提として参照させるための全体方針書である。

機能別の詳細指示に入る前に、以下を共有することを目的とする。

```text
saisoku が何のためのツールか
今回の優先機能は何か
設計上重視することは何か
今回やること・やらないことは何か
外部情報や画像をどう扱うか
Supabase Free前提でどこまで考えるか
```

---

## 2. saisoku の位置づけ

saisoku は、グランブルーファンタジーの団内運用を補助するための、身内向けWebツールである。

公開範囲はごく限られた身内・団内を想定しており、商用利用を目的としない。

攻略サイトそのものを作るのではなく、外部攻略サイト・動画・団内実績に分散している情報を、団内で使いやすい形に整理することを目的とする。

---

## 3. 今回の優先機能

今回優先して開発するのは、**高難易度向け編成共有機能** である。

この機能では、団内で以下を共有できるようにする。

```text
高難易度クエスト向けの編成
レヴァンス系の周回編成
必要キャラ
必要召喚石
必要武器
必須パーツ
代用候補
行動メモ
周回条件
注意点
参考URL
```

## 6. 操作思想

saisoku では、操作数・入力数を減らすことを重視する。

そのため、以下の方針を取る。

```text
毎回ゼロから入力しない
プリセットから反映して、変更したい箇所だけ修正する
候補リストからキャラ・召喚石・武器を選択する
候補にないものは自由入力できる
既存投稿をコピーして新規投稿できる
詳細画面では重複情報を減らし、実用情報を優先する
```

---

## 7. 編成登録の基本方針

編成登録は、画像投稿中心にはしない。

キャラ・召喚石・武器は、saisoku側で用意した候補リストから選択する方式を基本とする。

候補に存在しないものは自由入力できるようにする。

画像は、編成スクショの投稿を中心にするのではなく、キャラ・召喚石・武器の小型サムネイルとして補助表示に使う。

---

## 8. プリセット機能の方針

プリセットは、団内で使い回すための編成テンプレートである。

投稿者は、編成投稿時にプリセットを選択できる。  
プリセットを選択すると、投稿フォームに初期値として内容が反映される。

反映後、投稿者は自分の手持ちや団内運用に合わせて、変更したい箇所だけ編集して投稿できる。

プリセットを使用せず、空のフォームから投稿することもできる。

既存投稿をコピーして新規投稿する導線も用意する。

---

## 9. プリセットの由来と状態

プリセットには、由来を設定できるようにする。

```text
攻略サイト参考
YouTube参考
X参考
note / ブログ参考
団内実績
投稿者作成
その他
```

プリセットには、状態を設定できるようにする。

```text
参考用
未検証
投稿者確認済
団内利用中
要更新
```

外部情報を参考にする場合でも、本文・画像・表を丸ごと保存するのではなく、参考URLと団内向けメモとして扱う。

## 11. 編成パーツの扱い

編成パーツとして以下を扱う。

```text
主人公ジョブ
キャラ
召喚石
武器
```

各パーツには重要度を設定できるようにする。

```text
必須
推奨
代用可
自由枠
```

武器については、MVPでは完全な10枠再現を必須にしない。  
重要武器・必須武器・代用しづらい武器を中心に登録できればよい。

## 12. 一覧・検索画面の方針

編成一覧・検索画面は、団員が目的の編成をすぐ見つけるための画面である。

一覧では、まず以下を切り替えられるようにする。

```text
すべて
高難度攻略用
周回・武器集め用
```

絞り込み対象：

```text
クエスト名
属性
目的
操作タイプ
検証状態
投稿種別
キーワード
```

キーワード検索対象：

```text
編成タイトル
概要メモ
注意点
行動メモ
キャラ名
召喚石名
武器名
主人公ジョブ
参考URLタイトル
投稿者名
```

一覧カードでは、詳細を見るか判断するために必要な情報を表示する。

詳細画面では、一覧カードで確認済みの情報を重複して大きく表示しない。

---

## 13. 詳細画面の方針

編成詳細画面は、一覧カードから遷移する前提で設計する。

詳細画面は、分類情報を確認する画面ではなく、実際にその編成を使えるか確認するための画面とする。

詳細画面で重視する情報：

```text
必須パーツ
推奨パーツ
代用候補
キャラ・召喚石・武器の詳細
行動メモ
失敗ポイント
周回条件
注意点
参考URL
コピー・編集導線
```

詳細画面上部は以下に絞る。

```text
編成タイトル
主要ラベル
概要メモ
コピー・編集ボタン
```

クエスト名、分類、属性、目的、操作タイプなどは、必要に応じてタグ形式で簡潔に表示する。

## 14. 画像の扱い

画像は外部URL参照ではなく、saisoku側で自前保持する。

ただし、高解像度画像や編成スクショの大量保存は行わない。

画像は、キャラ・召喚石・武器の小型サムネイルとして扱う。

### 画像方針

```text
小型サムネイルのみ
1枚あたり50KB前後を目標
初期保存容量は300MB以内を目標
Supabase Freeの範囲に収める
外部画像URLへ直接依存しない
```

---

## 15. Supabase Free前提

現在の運用は Supabase Free を前提とする。

そのため、Storage容量とEgressを意識し、画像や一覧表示を軽量にする。

```text
画像は小型化する
一覧で大量画像を一度に表示しない
高解像度画像を保存しない
画像総数は初期800〜1,200枚程度を想定する
初期ストレージ使用量は40〜100MB程度を目安とする
```

Pro移行は、利用者数や画像表示量が増えた段階で検討する。  
現時点では Pro 移行を前提にしない。

## 17. マスタ・画像準備の方針

キャラ・召喚石・武器マスタの初期作成は、完全手作業ではなく半自動化する。

GBF Wiki / GBF Asset Lookup などの外部公開情報を参考に、以下を収集する。

```text
名称
属性
カテゴリ
参照URL
画像候補
```

ただし、saisoku本体の通常機能として、外部サイトを常時スクレイピングしない。

初期データ準備用の補助スクリプトとして扱い、取得後は人間が確認してから取り込む。

画像は小型サムネイル化して、saisoku側で自前保持する。

---

## 18. 外部情報の扱い

GameWith、神ゲー攻略、GBF Wiki、YouTube、X、note、ブログなどは参考情報として扱う。

saisoku内では、外部サイトの本文・画像・表を丸ごと保存しない。

保存するのは以下を基本とする。

```text
参考URL
参照元メモ
団内向けに整理したメモ
代用候補
注意点
検証メモ
```

## 21. Codexに依頼する際の進め方

Codexには、まずこの全体方針書を共有する。

そのうえで、機能別指示書を個別に渡す。

想定する機能別指示書は以下。

```text
01_高難易度編成共有_全体要件
02_編成プリセット機能
03_編成投稿フォーム
04_編成一覧・検索画面
05_編成詳細画面
06_キャラ石武器マスタ・サムネイル
07_参考URL・外部情報管理
08_マスタ初期作成補助スクリプト
```

1つの巨大な指示書でまとめて依頼するのではなく、機能単位で分割して進める。

## 22. 全体方針として最も重要なこと

この開発で最も重視するのは以下。

```text
団内で実際に使えること
入力の手間を減らすこと
外部情報を団内用に整理すること
画像投稿中心ではなく、選択式で検索可能にすること
Supabase Freeで軽量に運用できること
```

以上を、saisoku の高難易度向け編成共有機能における全体方針とする。
