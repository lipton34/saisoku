目標ボード・目標連結・編成スクショ対応

新しい独立機能を増やすのではなく、既存の「目標」「編成」「武器マスタ」「討伐数」をつなぎ、 グラブルでやることが多すぎて混乱しやすい状態を整理するための仕様まとめ。

目的 今やることを整理する
中心機能 目標ボード
連結対象 編成・武器・討伐数
画像方針 スクショ投稿を限定解禁

1. 今回の結論

方針： 素材管理ツールではなく、「目標を中心に、必要な編成・武器本数・討伐数を整理するツール」に寄せる。

目標を整理する

「今やる」「次にやる」「後でやる」「保留」「完了」でカンバン管理する。

目標ボード 初心者向け

目標と連結する

目標に、関連編成・必要武器・討伐目標・サブタスクを紐づける。

編成連結 武器本数

スクショ投稿を許可する

選択式の編成登録は残しつつ、編成スクショも添付できるようにする。

入力負荷軽減 詳細画面で確認

2. 対象外にすること

今回は、何でも管理できる万能ツールにはしない。特に素材管理は深追いしない。

項目 方針 理由
素材数の細かい管理 対象外 グラブル内や既存外部ツールである程度管理できるため。
全キャラ・全武器の所持状況自動判定 対象外 入力負荷と管理範囲が大きくなりすぎるため。
ダメージ計算・最適編成算出 対象外 saisokuの目的は攻略計算ではなく、団内情報整理のため。
外部サイト本文・画像の丸ごと保存 対象外 参考URLと団内向けメモに留める。
素材管理を完全にやろうとすると、対象範囲が一気に広がる。 saisokuでは「武器本数」「討伐数」「編成準備」のように、既存機能とつながる情報を優先する。

3. 全体像

目標を中心にして、そこから編成・武器・討伐数・サブタスクへつなげる。

目標 古戦場150HELL編成を準備する
関連編成 手動編成 / フルオート編成
必要武器 必要本数と現在本数
討伐目標 自発 / 救援の回数
サブタスク 確認・作成・練習など

4. 目標ボード

初心者・復帰者が「何をすればいいかわからない」と感じる原因は、 やることが多いことよりも、優先順位が整理されていないことにある。 そのため、目標をカンバンで整理する。

目標に持たせる項目

項目 値 用途
状態 今やる / 次にやる / 後でやる / 保留 / 完了 カンバンの列を決める。
優先度 高 / 中 / 低 何から進めるべきか判断する。
負荷 軽い / 普通 / 重い 今すぐできることか、重い作業か判断する。

初心者向け ON / OFF 初心者が見るべき目標だけを絞り込む。

5. 目標詳細ページ

カンバンカードを押すと、目標詳細に遷移する。 目標詳細では、その目標を達成するために必要なものを確認する。

目標：シエテ剣を2本作る

優先度 中 負荷 重い 後でやる

必要武器

  • セッテ・ディ・スパーダ：0 / 2

討伐目標

  • シエテHL 救援：120 / 300
  • シエテHL 自発：10 / 30

関連編成

シエテHL 青箱狙い編成へ遷移できる。

サブタスク

  • 必要武器を確認する
  • 関連編成をコピーする
  • 討伐数を更新する

6. 編成スクショ対応

現在の選択式の編成作成機能は残す。 その上で、スクショを添付して詳細画面で確認できるようにする。

選択式で作成

検索性・目標連結・必要武器数の管理に向いている。

既存機能を維持 構造化データ

スクショで作成

入力が面倒なときや、手早く共有したいときに向いている。

入力負荷軽減 最大5枚

画像カテゴリ

画像には種類を持たせ、詳細画面で見やすくする。

キャラ 武器 召喚石 ジョブ・アビリティ 討伐結果 その他

編成詳細：火 150HELL 手動編成

基本情報・必要パーツ・行動メモに加えて、スクショを確認できる。

キャラ編成スクショ
武器編成スクショ
召喚石編成スクショ

画像制限

項目 仕様
枚数 1編成あたり最大5枚
形式 jpg / jpeg / png / webp
サイズ 1ファイル最大5MB
一覧表示 大きな画像は出さず、「画像あり：3枚」程度にする
詳細表示 カテゴリ別に表示し、クリックで拡大できると望ましい

7. フェーズ分け

Phase 1：編成スクショアップロード対応

既存の編成共有機能に、画像添付を追加する。

編成に画像を最大5枚まで添付できる。
画像カテゴリを設定できる。
Supabase Storageに保存する。
編成詳細で画像を確認できる。
編成編集で画像を追加・削除できる。
一覧には重い画像を出さず、画像あり表示に留める。

Phase 2：目標ボード・優先度管理

目標をカンバン形式で整理できるようにする。

今やる / 次にやる / 後でやる / 保留 / 完了 の列を作る。
優先度・負荷・期限・初心者向けを設定できる。
カードクリックで目標詳細へ遷移できる。
初心者向けのみ表示、完了非表示などで絞り込める。

Phase 3：目標と編成・武器・討伐数の連結

目標を中心に、必要なものを紐づける。

目標に関連編成を紐づける。
目標に必要武器と必要本数を設定する。
目標に討伐数を設定する。
目標にサブタスクを追加できる。
編成詳細から関連目標を確認できる。

8. 実装優先順位

順番 実装内容 理由
1 編成スクショアップロード 既存の編成機能に閉じており、単体でも価値が出る。
2 目標ボード 初心者が「今やること」を整理できる土台になる。
3 目標 ↔ 編成 目標詳細から必要な編成にすぐ遷移できる。
4 目標 ↔ 必要武器 素材ではなく、武器本数を管理できるようにする。
5 目標 ↔ 討伐数 / サブタスク 武器作成・ヒヒ掘り・高難度練習の進捗を管理できる。
推奨は、まず Phase 1：編成スクショ対応 から始めること。 その後、目標ボードを作り、最後に目標と編成・武器・討伐数を連結する。

9. DB・APIの大まかな追加対象

Phase 1

追加対象 概要
BuildImage 編成に添付された画像のパス・種類・順番・MIME typeなどを持つ。
画像アップロードAPI Supabase Storageへ保存し、DBにメタ情報を保存する。
画像削除API DBとStorageの削除を行う。

Phase 2

追加対象 概要
shared_goalsの拡張 boardStatus / priority / effort / dueDate / beginnerRecommended / sortOrder を追加する。
目標更新API 状態・優先度・負荷・期限・初心者向けを更新する。

Phase 3

追加対象 概要
GoalBuildLink 目標と編成を紐づける。
GoalRequiredItem 目標に必要な武器・必要本数・現在本数を持つ。
GoalRaidTarget 目標に対する討伐回数を持つ。
GoalSubTask 目標を小さな作業に分解する。

10. 最終的な利用イメージ

初心者・復帰者

  • 目標ボードで「今やる」だけを見る。
  • カードを押して、必要な編成や武器を確認する。
  • 編成詳細でスクショを見て真似する。
  • 終わったら目標を完了にする。

団内の情報整理担当

  • イベントや古戦場前に目標を作る。
  • 関連編成を紐づける。
  • 必要武器や討伐目標を設定する。
  • 初心者向けラベルで見るべきものを絞る。
最終的に目指すのは、攻略情報を増やすことではなく、 「何を見ればいいか」「何からやればいいか」を整理して見せること。

11. Codexに実装依頼するときの注意

  • AGENTS.mdがあれば必ず確認する。
  • 既存のPrismaモデル名・APIルート・画面構成を確認してから実装する。
  • 既存の編成投稿・目標一覧・提案機能を壊さない。
  • 大きな変更はdocsに設計メモを追加する。
  • 画像は一覧で重く表示しない。
  • 素材管理や完全自動判定に広げない。
  • まずはPhase 1、その後Phase 2、最後にPhase 3の順で進める。
