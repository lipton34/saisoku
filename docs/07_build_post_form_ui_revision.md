# 目的

現在の投稿フォームは縦長の入力ページになっているが、入力項目が多く、キャラ・武器・召喚石の選択体験もグラブルらしさが弱い。

投稿フォームを、グラブル本家の編成画面に近い操作感で、段階的に入力していくUIに変更する。

特に、キャラ・召喚石・武器はサムネイル画像付きで選択できるようにし、検索可能なUIにする。

# 修正方針

## 1. 投稿フォームをステップ形式に変更する

現在の縦長フォームをやめ、以下の順番でページ遷移またはステップ切り替えしながら入力する形式に変更してください。

1. 基本情報入力
2. 主人公ジョブ + キャラ選択
3. 武器選択
4. 召喚石選択
5. その他情報入力

各ステップの役割は以下です。

## Step 1: 基本情報入力

編成の基本条件を入力する。

対象項目の例：

- 編成タイトル
- クエスト分類
- クエスト名
- 属性
- 目的
- 操作タイプ
- 検証状態
- 概要メモ

## Step 2: 主人公ジョブ + キャラ選択

主人公ジョブとキャラを選択する。

キャラ選択UIは、グラブル本家の編成画面に近い見た目・操作感を目指してください。

要件：

- 主人公ジョブを選択できる
- フロントキャラ枠を選択できる
- サブキャラ枠を選択できる
- キャラ候補はサムネイル画像付きで表示する
- キャラ名で検索できる
- 候補をクリックまたはタップして枠に反映できる
- 選択済みキャラを差し替えできる
- 候補に存在しないキャラは自由入力できる余地を残す

## Step 3: 武器選択

武器を選択する。

武器選択UIも、グラブル本家の武器編成画面に近い見た目・操作感を目指してください。

要件：

- 武器枠は10〜13枠を扱えるようにする
- 武器候補はサムネイル画像付きで表示する
- 武器名で検索できる
- 候補をクリックまたはタップして武器枠に反映できる
- 選択済み武器を差し替えできる
- 本数や重要度など、既存要件に必要な情報は維持する
- 候補に存在しない武器は自由入力できる余地を残す

## Step 4: 召喚石選択

召喚石を選択する。

召喚石選択UIも、グラブル本家の召喚石編成画面に近い見た目・操作感を目指してください。

要件：

- メイン召喚石枠を選択できる
- フレンド召喚石候補を選択できる
- サブ召喚石枠を選択できる
- 必要であればサブ加護枠も扱えるようにする
- 召喚石候補はサムネイル画像付きで表示する
- 召喚石名で検索できる
- 候補をクリックまたはタップして枠に反映できる
- 選択済み召喚石を差し替えできる
- 候補に存在しない召喚石は自由入力できる余地を残す

## Step 5: その他情報入力

その他の情報を入力する。

対象項目の例：

- 必須パーツ
- 推奨パーツ
- 代用候補
- 注意点
- 行動メモ
- 周回条件
- 参考URL
- プリセット由来や変更メモ

必須パーツなどについては、前段階で選択済みのキャラ・召喚石・武器を選択肢として表示してください。

例：

Step 2〜4で選択したキャラ・召喚石・武器
↓
Step 5の「必須パーツ」「推奨パーツ」「代用可」などの候補として表示する

ただし、選択肢化できないものについては、現在のテキスト入力方式を維持して構いません。

# キャラ・武器・召喚石選択UIについて

キャラ・武器・召喚石の選択UIは、それぞれタブで切り替えられるようにしてください。

例：

- キャラ
- 武器
- 召喚石

または、ステップ内で必要に応じて以下のように切り替えられるようにしてください。

- 選択済み枠
- 候補一覧
- 検索結果

選択UIでは以下を満たしてください。

- サムネイル画像を表示する
- 名称を表示する
- 属性や種別が分かるようにする
- 検索できる
- 候補選択時に対象枠へ反映する
- 選択済み枠の差し替えができる
- UIが動的に変化してもレイアウトが崩れないようにする

# プリセットUIの修正

現在、画面右側にプリセットが常時表示されているが、この表示は不要です。

プリセットは、専用ボタンから選択する方式に変更してください。

## 要件

- 投稿フォーム内に「プリセットを使う」ボタンを配置する
- ボタン押下時にモーダルを開く
- モーダル内に現在存在するプリセットをカード形式で表示する
- プリセットカードの表示内容は、現在のカード型表示をベースにしてよい
- ユーザーがプリセットを1つ選択すると、その内容を投稿フォームへ反映する
- 反映後も、ユーザーが各項目を自由に編集できるようにする
- 画面右側の常時表示プリセット欄は削除する

# レイアウト崩れの修正

現在、UIが動的に変化する場面で画面崩れが見られるため、あわせて修正してください。

特に確認してほしい箇所：

- ステップ切り替え時
- キャラ・武器・召喚石候補の検索時
- サムネイル一覧表示時
- プリセットモーダル表示時
- モーダルを閉じた後
- 選択済み枠が増減する場面
- スマホ幅または狭い画面幅

# 維持してほしい方針

以下は既存方針を維持してください。

- 画像投稿中心にはしない
- キャラ・召喚石・武器は候補リストから選択する方式を基本にする
- 候補にないものは自由入力できる
- サムネイルは軽量な補助表示として扱う
- 外部画像URLに直接依存しない
- Supabase Free前提で重くしすぎない
- 既存のReact + Vite / Express + TypeScript / Prisma構成を崩さない
- 既存の認証・JWT + cookieログイン実装と矛盾しない
- 要件書に沿ったコード変更は追加確認なしで実施する
