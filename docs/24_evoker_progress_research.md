# 十賢者進捗プリセット 調査結果

- 調査日: 2026-07-23
- 対象: `/progress-goals` の十賢者プリセット
- 関連仕様: `docs/23_progress_preset_feature.md`
- 状態: 調査・表示名称・素材ID確定、実装データ化待ち

素材マスター、`itemKey`、DB・version互換性の調査結果は `docs/25_progress_item_master_db_research.md` を参照する。

## 1. この文書の目的

十賢者の加入から最終上限解放、4アビ取得までを中継点へ分解し、各中継点の必要素材、依存関係、対象別置換に必要な情報を整理する。

この文書は調査結果であり、振る舞いの正本ではない。画面、計算、保存、互換性の仕様は `docs/23_progress_preset_feature.md` を優先する。

2026年7月に追加された十賢者の限界超越と「ソロモナスの賢者」は今回のプリセット対象外とする。初期ゴールは従来どおり「4アビ取得」までとする。

## 2. 結論

以下は中継点へ分解できるだけの段階別情報を確認できた。

- アーカルム召喚石: 交換、SR1凸、SR2凸、SR3凸、SSR化、SSR4凸、SSR5凸
- 加入: 専用カード交換素材
- 新世界の礎: 交換、1凸、2凸、3凸、4凸、5凸
- 至賢の領域: 第1～第4解放
- 最終上限解放: 最終上限解放素材
- 4アビ: 4アビ取得用素材

素材値はGameWithを基準情報、神ゲー攻略を照合先として確認する。両サイトが一致する値は採用し、不一致の場合だけゲーム内表示などで追加確認する。

数量はGameWithと神ゲー攻略で照合できた。既存の汎用GBFマスターは素材種別を扱え、安定した `itemKey` も `docs/25_progress_item_master_db_research.md` で確定した。残るのは、素材マスターseedとプリセット定義を実装し、転記値を機械的に再照合する作業である。

### 4.1 表示名称の確定方針

GameWithと神ゲー攻略では、同じ段階について「作成」「交換」「強化」などの表現が混在する。進捗ツリーはゲーム画面のメニュー名を再現する機能ではないため、表示名称は現在地を短く判別できる次の表記へ統一する。

| 工程グループ | 表示順 |
| --- | --- |
| アーカルム召喚石 | SR召喚石交換 → SR1凸 → SR2凸 → SR3凸 → SSR化 → SSR4凸 → SSR5凸 |
| 十賢者加入 | 十賢者加入 |
| 新世界の礎 | 礎武器交換 → 礎武器1凸 → 礎武器2凸 → 礎武器3凸 → 礎武器4凸 → 礎武器5凸 |
| 至賢の領域 | 第1解放 → 第2解放 → 第3解放 → 第4解放 → 全解放 |
| 最終上限解放 | 最終上限解放 |
| 4アビ取得 | 4アビ取得 |

表示上の補足:

- 「SR召喚石交換」は、初期状態がSRでありショップ交換から始まることを明示する。
- 「十賢者加入」には専用カード交換素材を含めるが、カード交換を独立中継点にはしない。
- 至賢の領域では、カード内に工程グループ名が併記されるため、各中継点を「至賢の領域 第1解放」のように重複表示しない。
- 「全解放」は素材を持たない集約目標であり、操作可能な通常中継点と視覚的に区別する。
- 内部IDは表示名称から生成しない。将来表示文言を変更しても保存済み進捗へ影響させない。

### 4.2 依存関係に関する根拠

- 十賢者加入には対応するアーカルム召喚石5凸と専用カード交換が必要。専用カード交換素材は `recruited` に登録する。
- 礎武器交換は対応する十賢者加入後に解禁。
- 至賢の領域は、礎武器入手で第1解放、1凸で第2解放、2凸で第3解放、3凸で第4解放まで進められる。
- 最終上限解放の素材計算では、礎武器5凸と至賢の領域全解放を前提中継点とする。
- 4アビ取得の素材計算では、最終上限解放を前提中継点とする。

討伐、Lv、フェイトエピソードなどの素材以外の前提はゲーム上必要だが、この機能の主目的である不足素材の算出には影響しない。十賢者の初期プリセットでは入力・保存対象にせず、関連中継点の補足説明だけで案内する。

## 9. 最終上限解放

### 9.1 ゲーム内前提条件（参考表示のみ）

- エリア・ムンドゥスのディフェンダーをすべて討伐

## 12. 調査完了後に実装で確認する事項

### 12.1 実装・利用可能化時の検証

- 実装データへ転記した全数量と本資料の機械的な再照合

旧・新召喚アニマ、討滅戦素材、石片、島素材の正式名称と対象別対応は `docs/25_progress_item_master_db_research.md` で確認済みである。
中継点の表示名称は第4.1節のプロダクト表記へ統一するため、ゲーム内UIとの追加照合は利用可能化の必須条件にしない。
素材の固定 `itemKey` も同資料の第10.4節で確定済みである。ここに残る項目は設計上の未決事項ではなく、実装データを作成した後に行う検証である。

### 12.2 確定したプロダクト仕様

- エリア・ムンドゥスとザ・ワールドの討伐状態は管理しない
- 加入、最終、4アビのフェイトエピソードは独立中継点や入力条件にしない
- `Lv100` は独立中継点や入力条件にしない
- 素材以外の前提は必要に応じて補足説明だけを表示する
- ゲーム内緩和アイテムやお助けセットを代替ルートとして扱わず、標準素材だけを表示する

## 13. 実装データへ変換する際の注意

- 情報源に合計値しかない場合でも、今回確認した個別段階へ再配分しない。個別表の値を直接登録する。
- 光・闇の分割素材を単一の抽象素材キーで保存しない。具体的な2つの `itemKey` へ解決する。
- 光・闇の分割数には単純な端数丸めを使わず、段階別の確定値を登録する。礎武器交換はブライト各3、1凸は各7で、通常属性の合計値より1多い・少ない特殊値になる。
- `domain-complete` のような集約目標に素材を登録しない。
- `final-uncap` は礎武器5凸と領域全解放の双方へ依存させる。討伐・フェイト条件は依存関係に含めない。
- `fourth-ability` は `final-uncap` に依存させ、4アビ取得用素材を直接登録する。Lv100と関連エピソードは依存関係に含めない。
- 完了済み中継点は計算から除外するが、ユーザーの所持数は自動減算しない。
- 未確認値が1件でも残る定義は `isAvailable: false` を維持する。
- 2026年追加の十賢者限界超越を、既存の4アビ取得ゴールへ混在させない。
