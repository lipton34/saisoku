# Codex調査指示書：マスタデータ管理の現状調査

AGENTS.md の指示に従ってください。

今回は実装変更ではなく、現状調査のみをお願いします。

目的は、現在の saisoku における「キャラ・武器・召喚石などのマスタデータ管理」がどうなっているかを把握し、今後 Supabase DB 管理へ移行すべきか判断するための材料を整理することです。

コード変更・ファイル修正・リファクタリングはまだ行わないでください。  
調査結果だけを報告してください。

---

## 1. 調査してほしい背景

現状、キャラ・武器・召喚石の候補は `src/lib/buildMasters.ts` の静的 TypeScript 配列で管理されているように見えます。

一方で、今後は素材・クエスト・画像パスなども増える想定があるため、マスタデータは Supabase DB のレコードとして管理した方がよいのではないかと考えています。

ただし、現在の実装が実際にどうなっているかを正確に把握できていないため、まず調査して認識を揃えたいです。

- 候補一覧はクリック時にどのデータを参照しているか
- クリック時にDBやAPIへ問い合わせているか

- `thumbnailUrl` の解決処理は毎回走っていないか
- 候補検索やフィルター処理は `useMemo` されているか

### 3.4 Supabase DBの現在の使われ方

以下を確認してください。

- 素材やクエスト用のテーブルは存在するか
- 既存のテーブル名・カラムが分かれば整理する

### 3.5 投稿保存データの構造

以下を確認してください。

- 自由入力したパーツはどう保存されているか
- 将来DBマスタ化した場合、既存投稿データとの互換性に問題がありそうか

### 3.6 DBマスタ化する場合の影響範囲

以下を調査してください。

- `buildMasters.ts` をDB取得に置き換える場合、影響するファイル
- 投稿フォームで変更が必要な箇所
- 一覧・詳細画面で変更が必要な箇所
- 既存投稿保存形式を変えずに移行できそうか
- `buildMasters.ts` を fallback として残せるか
- クリック時にDB問い合わせせず、初期取得後にフロントでキャッシュする構成にできそうか
