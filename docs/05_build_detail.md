# Codex作業指示：05_編成詳細画面

## 1. 目的

`saisoku` に、高難易度向け編成共有機能の一部として **編成詳細画面** を追加・整備する。

編成詳細画面は、一覧カードから遷移する前提で設計する。

この画面の目的は、分類情報を再確認することではなく、実際にその編成を使えるか確認し、必要ならコピーして流用することである。

---

作業前に、必ず以下を確認すること。

```text
docs/requirements/01_overall_policy.md
```

## 3. 前提となる導線

編成詳細画面は、基本的に以下の導線で到達する。

```text
編成一覧・検索画面
↓
クエスト分類・クエスト名・属性・目的などで絞り込み
↓
カード形式で編成候補を確認
↓
気になる編成カードを選択
↓
編成詳細画面へ
```

このため、詳細画面に到達した時点で、ユーザーは以下をある程度把握している。

```text
どのクエスト向けか
どの属性か
高難度攻略用か、周回・武器集め用か
手動かフルオートか
未検証か、クリア済みか
```

したがって、詳細画面では一覧カードで確認済みの情報を大きく繰り返さない。

---

## 4. 一覧カードと詳細画面の役割分担

### 一覧カードの役割

一覧カードは、**詳細を見るかどうかを判断するための画面要素** とする。

一覧カードで表示する主な情報：

```text
編成タイトル
属性
目的
操作タイプ
検証状態
主要パーツ
参考URL有無
行動メモ有無
更新日
```

### 詳細画面の役割

詳細画面は、**実際にその編成を使えるか確認し、必要なら流用するための画面** とする。

詳細画面で重視する情報：

```text
必須パーツ
推奨パーツ
代用候補
キャラ・召喚石・武器の詳細
行動メモ
失敗ポイント
周回条件
注意点
参考URL
コピー・編集導線
```

---

## 5. 詳細画面の基本方針

```text
クエスト名・分類・属性・目的・操作タイプなどを文章で重複表示しない
必要な情報はタグ形式で簡潔に表示する
詳細画面では実践に必要な情報を優先する
必須パーツと代用候補を早い位置で確認できるようにする
```

---

## 6. 詳細画面上部の表示

詳細画面の上部には、以下を表示する。

```text
編成タイトル
主要ラベル
概要メモ
コピー・編集ボタン
```

主要ラベルはタグ形式で表示する。

例：

```text
[水] [手動] [団内練習] [投稿者クリア済]
```

レヴァンス系の例：

```text
[火] [フルオート] [自発向け] [安定] [神石]
```

### 上部に大きく表示しないもの

以下は、詳細画面上部で大きく表示しない。

```text
クエスト分類
クエスト名
属性
目的
操作タイプ
投稿種別
投稿者
更新日
```

ただし、必要に応じて小さな補足表示やタグとして扱う。


## 9. 必須パーツ表示

詳細画面では、編成パーツ全体よりも先に **必須パーツ** をまとめて表示する。

目的：

```text
この編成を組むために絶対に必要なものをすぐ確認する
代用しにくい要素を把握する
自分の手持ちと照らし合わせやすくする
```

表示対象：

```text
必須キャラ
必須召喚石
必須武器
```

表示項目：

```text
サムネイル
名称
種別
重要度
理由メモ
```

---

## 10. 代用候補表示

必須パーツの次に、代用候補を表示する。

表示項目：

```text
元パーツ
代用候補
代用時の注意点
安定度・速度への影響
```

例：

```text
キャラA → キャラBでも可
ただし予兆解除が不安定になるため、召喚石Cで補助する
```

---

## 11. 編成パーツ表示

編成パーツは以下に分けて表示する。

```text
主人公ジョブ
キャラ
召喚石
武器
```

### キャラ

配置：

```text
フロント
サブ
任意
```

表示項目：

```text
サムネイル
キャラ名
配置
重要度
役割メモ
代用メモ
```

### 召喚石

配置：

```text
メイン
フレンド
サブ
任意
```

表示項目：

```text
サムネイル
召喚石名
配置
重要度
用途メモ
代用メモ
```

### 武器

武器は完全な10枠再現を必須にしない。

MVPでは、重要武器・必須武器・代用しづらい武器を中心に表示する。

表示項目：

```text
サムネイル
武器名
本数
重要度
用途メモ
代用メモ
```

---

## 12. 行動メモ表示

高難度攻略用では、行動メモを重要項目として表示する。

見出し：

```text
開幕
100〜80%
80〜60%
60〜20%
20〜0%
予兆対応
失敗ポイント
その他メモ
```

すべての見出しを常に表示する必要はない。入力されている項目だけ表示する。

---

## 13. 周回メモ表示

周回・武器集め用では、行動メモよりも周回条件・注意点を優先する。

表示項目：

```text
自発 / 救援
青箱狙い
討伐時間目安
安定度
編成前提
武器集め対象
救援タイミング
周回時の注意点
```

---

## 14. 参考URL表示

参考URLは、詳細画面の下部にまとめて表示する。

URL種別：

```text
攻略記事
YouTube
X
note / ブログ
その他
```

表示項目：

```text
タイトル
種別
URL
メモ
```

MVPでは、URL登録・表示を優先する。YouTubeの画面内再生はMVP対象外とする。

---

## 15. プリセット由来の表示

プリセットから作成された編成の場合、必要に応じて以下を表示する。

```text
元にしたプリセット名
プリセットの由来
変更メモ
```

ただし、上部に大きく出す必要はない。詳細下部または補足エリアに表示する。

---

## 16. コピー・編集導線

詳細画面には、以下の導線を用意する。

MVPで入れる：

```text
この編成をコピーして新規投稿
編集する
```

コピー機能は、プリセット機能と同じく入力数削減に直結するため優先度が高い。

将来検討：

```text
この編成をプリセット化する
更新履歴を見る
コメントする
```

以下ができれば、この機能のMVPは完了とする。

```text

周回・武器集め用では周回条件や注意点を確認できる
参考URLを確認できる
編成をコピーして新規投稿できる
```
