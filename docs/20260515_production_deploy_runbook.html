Saisoku 本番環境デプロイ手順書

React + Vite / Express + TypeScript / Prisma / Supabase PostgreSQL / Render Free 構成の本番反映手順です。

対象: https://saisoku.onrender.com Repo: lipton34/saisoku 作成日: 2026-05-15

1. 前提

  • 本番環境は Render Web Service で稼働している。
  • DB は Supabase PostgreSQL を使用し、DATABASE_URL は Supabase Session pooler URL を使う。
  • 本番の秘密情報は GitHub に置かず、Render の Environment Variables に設定する。
  • Render の Build Command は npm install --include=dev && npm run prisma:generate && npm run build。
  • Render の Start Command は npm run start。
注意: Prisma migration がある変更は、コード反映前に本番DBへ npm run prisma:deploy を実行する段取りを必ず確認してください。

2. デプロイ前チェック

  • 作業ブランチまたは main の変更内容を確認する。
  • .env や秘密情報が差分に含まれていないことを確認する。
  • 破壊的なDB変更、認証/JWT/cookieまわりの変更、環境変数追加がないか確認する。
  • DB schema を変えた場合は migration ファイルが作成されていることを確認する。
git status --short
git diff --stat
git diff -- . ':(exclude).env'
今回のようなフロントUI中心の変更で Prisma schema や migration に変更がなければ、本番DB migration は不要です。

3. ローカル検証

最低限、型チェックと本番ビルドを通します。

npm install
npm run typecheck
npm run build

必要に応じてローカル起動し、画面操作を確認します。

npm run dev
確認項目 見る場所
ログイン / 新規登録 http://localhost:5173
APIヘルスチェック http://localhost:4000/api/health
編成投稿フォーム /builds の「投稿フォーム」タブ

4. DB migration がある場合

Prisma schema を変更した場合だけ実施します。ローカルで migration を作成済みであることを確認してください。

npm run prisma:generate
npm run prisma:migrate -- --name <migration_name>

本番DBへ適用するときは、Render の Shell か、Render と同じ本番 DATABASE_URL を設定した環境で次を実行します。

npm run prisma:deploy
重要: 本番DB向けに prisma:migrate は使わず、作成済み migration を適用する prisma:deploy を使います。

5. GitHub へ反映

Render が GitHub の対象ブランチを監視している場合、push 後に自動デプロイが始まります。

git status --short
git add .
git commit -m "Update build post form UI"
git push

コミットメッセージは変更内容に合わせて変えてください。ドキュメントのみなら例:

git commit -m "Add production deploy runbook"

6. Render 側の設定確認

項目 値
Build Command npm install --include=dev && npm run prisma:generate && npm run build
Start Command npm run start
Instance Type Free
Environment NODE_ENV=production, DATABASE_URL, JWT_SECRET, 必要なら INVITE_CODE
Render Free は一定時間アクセスがないとスリープします。デプロイ直後や初回アクセス時は起動待ちが発生することがあります。

7. デプロイ実行

  1. GitHub に push する。
  2. Render Dashboard で対象サービスを開く。
  3. Deploy が開始されたことを確認する。
  4. Build log で npm run prisma:generate と npm run build が成功していることを確認する。
  5. Deploy log で npm run start が起動していることを確認する。

自動デプロイが動かない場合は、Render Dashboard から Manual Deploy を実行します。

8. 本番動作確認

公開URL:

https://saisoku.onrender.com

APIヘルスチェック:

curl -sS -L https://saisoku.onrender.com/api/health
  • トップページが表示される。
  • ログインできる。
  • 既存データが表示される。
  • 編成一覧・検索が表示される。
  • 投稿フォームを開ける。
  • プリセットモーダルを開閉できる。
  • キャラ・武器・召喚石の候補検索と選択ができる。
  • スマホ幅または狭い画面幅でレイアウトが崩れない。

9. 問題発生時の切り戻し

Render の前回デプロイへ戻す

  1. Render Dashboard で対象サービスを開く。
  2. Deploys から直前の安定版 Deploy を選ぶ。
  3. Rollback / Redeploy を実行する。

Git の戻しコミットを作る

git log --oneline -5
git revert <problem_commit_sha>
git push
DB migration を本番適用済みの場合、コードだけ戻してもDB構造が戻らないことがあります。migration を含む変更では、事前に戻し方を別途決めてください。
