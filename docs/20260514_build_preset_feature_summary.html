Saisoku Feature Summary

編成プリセット機能 変更まとめ

高難易度向け編成共有機能の一部として、プリセットを選んで投稿フォームへ反映し、必要な部分だけ編集して編成メモを投稿できる導線を追加した。

高難度攻略用 周回・武器集め用 プリセット反映 コピー投稿 参考URL管理

今回追加したもの

編成プリセット画面

新しく /builds を追加した。サイドバーとホーム画面の「編成メモ」から移動できる。

プリセットカード

プリセット名、属性、目的、操作タイプ、検証状態、プリセット状態、主要パーツ、参考URL、更新日を確認できる。

投稿フォームへの初期値反映

プリセットを選ぶと、クエスト情報、キャラ、召喚石、武器、必須パーツ、注意点、行動メモ、参考URLなどがフォームへ入る。

既存投稿のコピー

投稿済み編成のコピーアイコンから、内容を新規投稿フォームへ反映できる。元プリセット名と変更メモも保持できる。

使い方

  1. ログイン後、サイドバーの「編成メモ」を開く。
  2. 投稿フォームで「クエスト分類」「クエスト名」「属性」を選ぶ、または入力する。
  3. 右側のプリセット候補から使いたいカードを選ぶ。
  4. フォームに反映された内容を、自分の手持ちや団内運用に合わせて編集する。
  5. 必要に応じて「変更メモ」を書く。
  6. 参考URLは 種別 | タイトル | URL | メモ の形式で1行ずつ入力する。
  7. 「編成メモを投稿」を押す。

プリセットは完成品ではなく初期値として扱う。反映後のキャラ、召喚石、武器、目的、検証状態、概要、行動メモ、参考URLは自由に編集できる。

プリセットを使わない投稿

画面上部の「空フォーム」を押すと、プリセットなしの新規投稿状態に戻る。特殊な編成や、まだプリセットがないクエストのメモは空フォームから登録できる。

既存投稿をコピーする流れ

  1. 右側の「投稿済み」からコピーしたい編成を探す。
  2. コピーアイコンを押す。
  3. タイトルが「コピー」付きになり、内容が投稿フォームに反映される。
  4. 変更したい箇所だけ編集して投稿する。

保存する情報

  • 共通情報: 編成タイトル、分類、クエスト名、属性、目的、操作タイプ、検証状態、概要メモ
  • 編成パーツ: 主人公ジョブ、キャラ、召喚石、武器、必須、推奨、代用可、自由枠
  • 高難度用: 役割、予兆対応、行動メモ、失敗ポイント
  • 周回用: 周回目的、自発 / 救援、青箱狙い、討伐時間目安、周回時の注意点
  • 参照情報: 元プリセット名、変更メモ、複数の参考URL

外部情報の扱い

要件に合わせて、外部攻略サイトの本文・画像・表を丸ごと保存する実装にはしていない。保存するのは参考URL、参照元メモ、団内向けに整理したメモ、代用候補、注意点、検証メモに限定する。

主な変更ファイル

  • src/pages/BuildsPage.tsx: 編成プリセット画面
  • server/routes/builds.ts: 編成プリセット・編成投稿 API
  • prisma/schema.prisma: BuildPost モデル追加
  • prisma/migrations/20260514112000_add_build_posts/migration.sql: DB migration
  • src/lib/api.ts: フロント用 API 型と関数
  • src/App.tsx, src/components/AppShell.tsx, src/pages/HomePage.tsx: 導線追加
  • src/styles.css: 編成プリセット画面のスタイル

デプロイ時の注意

今回は BuildPost テーブルを追加しているため、本番DBに migration を適用する必要がある。Render の自動デプロイだけで migration が走らない場合は、別途 npm run prisma:deploy を実行する。
