# Codex指示書：団内目標共有機能の実装

## 目的

saisoku に「団内目標共有機能」を追加してください。

この機能は、古戦場専用ではなく、団内で使う汎用的な個人目標共有機能です。
団員が自分の目標を設定し、進捗を手入力で更新できるようにします。
また、他の団員が目標を提案し、提案された本人が受け入れた場合のみ、その目標を自分の目標として登録できるようにしてください。

単なるノルマ管理ではなく、団内で「今誰が何を目標にしているか」「どこまで進んでいるか」を共有しやすくすることを重視してください。

## 実装対象

MVPとして、以下を実装してください。

```text
自分の目標作成
自分の目標進捗更新
団員全体の目標一覧表示
団員・カテゴリ・状態・期限・キーワードによるフィルター
目標詳細表示
他団員への目標提案
自分に届いた提案一覧
提案の受け入れ
提案の見送り
受け入れた提案から個人目標を作成
```

現在は1つの団での運用のみを想定します。
複数団管理や複雑な権限管理はMVPでは不要です。
全アカウントの目標は団内で確認できる前提で実装してください。

---

## 実装対象外

MVPでは、以下は実装しないでください。

```text
複数団管理
団ごとの所属管理
詳細な権限管理
管理者による他団員目標の強制編集
ゲーム内データの自動取得
貢献度や素材数の自動連携
SlackやDiscordへの通知
提案の取り下げ
提案へのコメント機能
目標の承認フロー
非公開目標
公開範囲の細かい設定
ランキング機能
達成バッジ機能
実績の自動集計
```

## 画面・導線

既存アプリの構成に合わせて、以下の画面または導線を追加してください。

| 画面         | 目的                                             |
| ------------ | ------------------------------------------------ |
| 目標一覧画面 | 団員全体の目標を一覧表示する                     |
| 目標詳細画面 | 目標の詳細と進捗を確認する                       |
| 目標作成画面 | 自分の目標を作成する                             |
| 進捗更新画面 | 自分の目標の現在値・メモ・状態を更新する         |
| 目標提案画面 | 他団員に目標を提案する                           |
| 提案一覧画面 | 自分に届いた提案を確認し、受け入れ・見送りを行う |

画面を完全に分離する必要はありません。
既存UIに合わせて、モーダル・タブ・セクション分割など、実装しやすく使いやすい形にしてください。

## 目標データの想定項目

既存のDB設計・命名規則に合わせて調整して構いませんが、概ね以下の項目を扱えるようにしてください。

| 項目                | 内容                           |
| ------------------- | ------------------------------ |
| id                  | 目標ID                         |
| user_id             | 目標を持つ団員のユーザーID     |
| title               | 目標タイトル                   |
| category            | 目標カテゴリ                   |
| description         | 目標内容                       |
| target_value        | 目標値                         |
| current_value       | 現在値                         |
| unit                | 単位                           |
| progress_rate       | 進捗率                         |
| status              | 目標状態                       |
| due_date            | 期限                           |
| memo                | メモ                           |
| source_proposal_id  | 提案から作成された場合の提案ID |
| proposed_by_user_id | 提案者のユーザーID             |
| created_at          | 作成日時                       |
| updated_at          | 更新日時                       |

`target_value` と `current_value` が入力されている場合は、進捗率を表示できるようにしてください。
数値目標がない場合は、進捗率を無理に表示せず、状態とメモで進捗を表現してください。

## 重要な操作仕様

### 自分で目標を作成する

```text
1. 目標作成画面または作成モーダルを開く
2. タイトル・カテゴリ・内容・期限・目標値などを入力する
3. 保存する
4. 目標一覧に表示される
```

### 自分の進捗を更新する

```text
1. 自分の目標を開く
2. 現在値・進捗メモ・状態を入力する
3. 保存する
4. 一覧と詳細に反映される
```

### 他団員に目標を提案する

```text
1. 目標提案画面または提案モーダルを開く
2. 提案先団員を選択する
3. 目標内容を入力する
4. 提案を保存する
5. 提案先団員の提案一覧に表示される
```

### 提案を受け入れる

```text
1. 自分に届いた提案を開く
2. 受け入れるを選択する
3. 提案内容を元に、自分の目標が作成される
4. 提案状態を「受け入れ済み」にする
5. 作成された目標IDを提案データに紐づける
```

### 提案を見送る

```text
1. 自分に届いた提案を開く
2. 見送るを選択する
3. 提案状態を「見送り」にする
```

## UI上の注意点

この機能はノルマ管理ではなく、団内の目標共有を目的とします。
そのため、圧が強い表現は避けてください。

避けたい表現：

```text
未達成者
遅れている団員
ノルマ未達
```

使いやすい表現：

```text
進行中
もう少し
達成済み
目標調整中
```

また、他団員からの提案は、本人が受け入れるまで個人目標にしないでください。
他人が勝手に目標を登録できる動きにはしないでください。

## 受け入れ条件

最低限、以下ができる状態にしてください。

```text
ログイン中ユーザーが自分の目標を作成できる
ログイン中ユーザーが自分の目標の進捗を更新できる
全団員の目標を一覧で確認できる
団員・カテゴリ・状態で目標を絞り込める
目標詳細を確認できる
他団員に目標を提案できる
自分に届いた提案を一覧で確認できる
提案を受け入れると、自分の目標として登録される
提案を見送ると、提案状態が見送りになる
数値目標がある場合は進捗率が表示される
数値目標がない場合でも、状態とメモで目標管理できる
```
